# 문서 포맷 가이드

문서 프로젝트의 **디렉토리 구조, 파일 위치, 네이밍 규칙**을 안내합니다.

---

## .ouroboros/docs/ 구조

```
.ouroboros/
├── docs/
│   ├── draft/          # /draft 산출물
│   ├── spec/           # /spec 설계 문서
│   ├── write/          # /write 본문
│   ├── qa/             # /review 리뷰 보고서
│   ├── knowledge/      # /research 조사 자료
│   └── todo/           # 작업 목록
├── context/
│   ├── current.md      # 현재 작업 상태 (View)
│   └── state.json      # 기계 읽기용 상태 (Primary)
├── env/
│   └── .env            # 서버 연결 설정
└── workflow/
    ├── protocols/      # 워크플로우 프로토콜
    └── templates/      # 문서 템플릿
```

---

## 파일 네이밍 규칙

| 디렉토리 | 패턴 | 예시 |
|---------|------|------|
| `draft/` | `YYYY-MM-DD-제목.md` | `2026-03-30-주간회의요약.md` |
| `spec/` | `YYYY-MM-DD-제목.md` | `2026-03-30-Q1마케팅보고서.md` |
| `write/` | `YYYY-MM-DD-제목.md` | `2026-03-30-Q1마케팅보고서.md` |
| `qa/` | `YYYY-MM-DD-제목-review.md` | `2026-03-30-Q1마케팅보고서-review.md` |
| `knowledge/` | `YYYY-MM-DD-주제.md` | `2026-03-30-SaaS시장동향.md` |

<Callout type="info">
  버전 관리가 필요한 문서는 파일명 끝에 `.v2`, `.v3`을 붙입니다.
</Callout>

---

## 소스 문서 위치

원본 자료(PDF, DOCX, XLSX 등)는 다음 위치에 저장합니다:

```
.ouroboros/docs/source/       # 원본 파일
.ouroboros/docs/source/data/  # 데이터 파일 (CSV, XLSX)
.ouroboros/docs/source/ref/   # 참고 문헌
```

---

## 문서 읽기 도구

다양한 포맷의 원본 문서를 Claude Code에서 읽으려면 `markitdown` 패키지를 설치합니다:

```bash
pip install markitdown pymupdf pdfminer.six mammoth openpyxl pandas python-pptx
```

지원 포맷:

| 포맷 | 확장자 |
|------|--------|
| PDF | `.pdf` |
| Word | `.docx` |
| Excel | `.xlsx`, `.csv` |
| PowerPoint | `.pptx` |
| 이미지 | `.png`, `.jpg` (OCR) |

## .gitignore 권장 설정

```gitignore
# 개인 설정 (커밋 금지)
.ouroboros/env/
.mcp.json

# 원본 대용량 파일 (선택)
.ouroboros/docs/source/*.pdf
.ouroboros/docs/source/*.xlsx
```

<Callout type="warning">
  `.ouroboros/env/.env`에는 사용자 인증 정보가 포함됩니다. 절대 Git에 커밋하지 마세요.
</Callout>
