# 문서 품질 기준

`/review` 스킬이 적용하는 **4대 문서 품질 기준**입니다.

---

## 1. 논리 구조

| 원칙 | 설명 | 체크 포인트 |
|------|------|-----------|
| **MECE** | 상호 배타적이고 전체를 포괄 | 섹션 간 중복/누락 없는지 |
| **피라미드** | 결론 먼저, 근거는 아래로 | 핵심 메시지가 첫 단락에 있는지 |
| **논증 흐름** | 주장-근거-결론 일관성 | 각 섹션이 다음 섹션으로 자연스럽게 연결되는지 |
| **범위 적절성** | 주제 대비 깊이/폭 균형 | 불필요하게 넓거나 지나치게 좁지 않은지 |

<Callout type="info">
  비즈니스 문서는 **피라미드 구조**(결론 → 근거 → 세부사항)가 가장 효과적입니다.
</Callout>

---

## 2. 근거 기반

| 항목 | 기준 |
|------|------|
| **출처 명시** | 모든 데이터와 인용에 출처 표기 |
| **신뢰도** | 1차 자료 > 2차 자료 > 의견. 출처 등급 구분 |
| **최신성** | 데이터 기준 시점 명시. 2년 이상 된 데이터는 경고 |
| **정량적 근거** | 주장에 수치를 동반 (가능한 경우) |
| **반론 고려** | 핵심 주장에 대한 반론/한계 언급 |

---

## 3. 가독성

| 항목 | 가이드 |
|------|--------|
| **문장 길이** | 한 문장 40자 내외. 복합문보다 단문 선호 |
| **단락 길이** | 한 단락 3-5문장. 긴 단락은 분리 |
| **시각 자료** | 표, 차트, 다이어그램 적극 활용. 본문 반복 금지 |
| **포맷 일관성** | 제목 체계, 번호 매기기, 볼드/이탤릭 규칙 통일 |
| **여백** | 적절한 줄 간격과 섹션 구분 |

---

## 4. 독자 맞춤

| 독자 유형 | 어조 | 주의점 |
|----------|------|--------|
| **경영진** | 간결, 결론 중심, 판단 요청형 | 기술 용어 최소화, 비즈니스 임팩트 강조 |
| **실무자** | 구체적, 실행 가능한 액션 중심 | 배경 설명 간략히, 실행 방법 상세히 |
| **기술팀** | 정확한 용어, 상세 스펙 | 주관적 표현 지양, 데이터 중심 |
| **외부 이해관계자** | 공식적, 신뢰감 있는 톤 | 내부 용어 배제, 자기 완결적 설명 |
| **일반 대중** | 쉬운 용어, 비유 활용 | 전문 용어 사용 시 반드시 설명 |
