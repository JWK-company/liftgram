# 대시보드 API

대시보드는 REST API를 제공합니다. 모든 요청에 `Authorization: Bearer <API_KEY>` 헤더가 필요합니다.

---

## 인증

모든 API 요청에 API 키를 포함합니다.

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://dashboard.example.com/api/memories
```

---

## 메모리 API

| 메서드 | 엔드포인트 | 설명 |
|--------|----------|------|
| `GET` | `/api/memories` | 메모리 목록 조회 |
| `GET` | `/api/memories/:id` | 메모리 상세 조회 |
| `POST` | `/api/memories/search` | 메모리 벡터 검색 |
| `PUT` | `/api/memories/:id` | 메모리 수정 |
| `DELETE` | `/api/memories/:id` | 메모리 삭제 |
| `POST` | `/api/memories/:id/archive` | 메모리 아카이브 |

### GET /api/memories

```bash
# 전체 목록
curl -H "Authorization: Bearer $API_KEY" \
  "https://dashboard.example.com/api/memories"

# 필터 적용
curl -H "Authorization: Bearer $API_KEY" \
  "https://dashboard.example.com/api/memories?type=gotcha&scope=project&status=active"
```

**쿼리 파라미터:**

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `type` | string | gotcha, pattern, decision, insight, domain-knowledge, preference |
| `scope` | string | project, personal |
| `status` | string | active, stale_candidate, archived |
| `limit` | number | 결과 수 제한 (기본 20) |
| `offset` | number | 페이지네이션 오프셋 |

### POST /api/memories/search

```bash
curl -X POST -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query": "보고서 데이터 인용 주의사항", "limit": 5}' \
  "https://dashboard.example.com/api/memories/search"
```

## Knowledge Graph API

| 메서드 | 엔드포인트 | 설명 |
|--------|----------|------|
| `GET` | `/api/knowledge/nodes` | 노드 목록 조회 |

### GET /api/knowledge/nodes

```bash
curl -H "Authorization: Bearer $API_KEY" \
  "https://dashboard.example.com/api/knowledge/nodes?type=writing_pattern&namespace=project"
```

## 응답 형식

모든 API는 JSON 형식으로 응답합니다.

```json
// 성공
{
  "success": true,
  "data": { ... },
  "meta": {
    "total": 100,
    "limit": 20,
    "offset": 0
  }
}

// 에러
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "Memory not found"
  }
}
```
