# 시스템 아키텍처

Ouroboros MCP 서버의 전체 아키텍처를 설명합니다.

---

## 전체 구성

```
┌─────────────────────────────────────────────────┐
│                 Claude Code                      │
│              (Template 설치됨)                    │
│   12 Skills / 6 Hooks / 동적 전문가 / 10 Protocols │
└───────────────┬─────────────────────────────────┘
                │ MCP (HTTP)
                ▼
┌─────────────────────────────────────────────────┐
│              Ouroboros MCP 서버                   │
│                                                  │
│   ┌──────────┐  ┌───────────┐  ┌──────────────┐ │
│   │ Memory   │  │ Knowledge │  │ Event/Session│ │
│   │ Service  │  │ Service   │  │ Service      │ │
│   └────┬─────┘  └─────┬─────┘  └──────┬───────┘ │
│        │              │               │          │
│   ┌────▼──────────────▼───┐    ┌──────▼───────┐ │
│   │       Neo4j           │    │  ClickHouse  │ │
│   │  (벡터 검색 + KG)     │    │  (이벤트 로그) │ │
│   └───────────────────────┘    └──────────────┘ │
│                                                  │
│   ┌──────────────┐    ┌────────────────────────┐ │
│   │   Ollama     │    │      Dashboard         │ │
│   │ (GPU 임베딩)  │    │    (웹 대시보드)        │ │
│   └──────────────┘    └────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

## 컴포넌트 상세

### Neo4j

| 항목 | 설명 |
|------|------|
| **역할** | 시맨틱 메모리 저장 + 지식 그래프 + 벡터 검색 |
| **데이터** | Memory 노드, 문서 구조, 프로젝트 지식, 사용자 선호 |
| **벡터 검색** | Ollama 임베딩(1024d)을 저장하고 유사도 검색 수행 |

단일 Neo4j 인스턴스에서 메모리와 지식 그래프를 모두 관리합니다.
네임스페이스로 프로젝트/사용자 데이터를 격리합니다.

### ClickHouse

| 항목 | 설명 |
|------|------|
| **역할** | 대화 기록 + 이벤트 로그 + 시계열 분석 |
| **데이터** | 대화 전량(JSONL), 도구 호출 기록, 세션 메트릭 |
| **특징** | 컬럼 지향 스토리지로 대량 로그 효율적 저장/조회 |

대시보드의 Sessions 탭에서 전체 대화 기록을 조회할 수 있습니다.

### Ollama

| 항목 | 설명 |
|------|------|
| **역할** | GPU 임베딩 생성 |
| **모델** | qwen3-embedding (1024 dimensions) |
| **용도** | 메모리 저장 시 벡터 임베딩, 검색 시 쿼리 임베딩 |

<Callout type="info">
  Ollama는 GPU를 활용하여 빠른 임베딩 생성을 수행합니다. GPU가 없으면 CPU fallback으로 동작합니다.
</Callout>

### MCP 서버

| 항목 | 설명 |
|------|------|
| **역할** | Claude Code와 백엔드를 연결하는 단일 진입점 |
| **프로토콜** | MCP over HTTP |
| **도구** | 18개 MCP 도구 제공 ([상세](/infra/mcp-tools)) |

### Dashboard

| 항목 | 설명 |
|------|------|
| **역할** | 웹 기반 관리/모니터링 대시보드 |
| **기능** | 메모리 조회, 세션 기록, 지식 그래프, 관리자 기능 |
| **인증** | API 키 (OUROBOROS_API_KEY) |

## 데이터 흐름

### 메모리 저장

```
Claude Code → memory_store → Ollama(임베딩) → Neo4j(저장)
```

### 메모리 검색

```
Claude Code → memory_search → Ollama(쿼리 임베딩) → Neo4j(벡터 검색) → RRF 리랭킹 → 결과
```

### 이벤트 기록

```
Claude Code → Hook(event-capture) → event_record → ClickHouse(저장)
```

### 대화 동기화

```
Claude Code → Hook(conversation-sync) → event_sync_conversation → ClickHouse(저장)
```

## 프로젝트/개인 메모리 분리

| 구분 | scope | 설명 |
|------|-------|------|
| **프로젝트 메모리** | `project` | 프로젝트 전체에서 공유. 문서 전략, 공통 패턴 |
| **개인 메모리** | `personal` | 특정 사용자 전용. 개인 선호, 작업 맥락 |

- 검색 시 양쪽 모두 포함
- 같은 시점이면 **개인 > 프로젝트** 우선
- 충돌 시 "프로젝트 지식과 다른 개인 기록이 있습니다" 1회 질문
