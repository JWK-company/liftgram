# 첫 사용

설치가 완료되었다면, Claude Code를 실행하고 워크플로우를 시작해봅니다.

---

## Claude Code 실행

```bash
claude
```

Claude Code가 시작되면 MCP 서버 목록에 `ouroboros`가 표시됩니다.
처음 연결할 때 Keycloak 인증이 필요합니다.

<Callout type="info">
  `/mcp` → `ouroboros` → **Authenticate** 선택 → 브라우저에서 Keycloak 로그인.
  인증 후 자동으로 연결되며, 토큰은 7일간 유지됩니다.
</Callout>

## /suggest로 시작하기

무엇을 해야 할지 모르겠다면 `/suggest`부터 시작하세요.

```
/suggest
```

`/suggest`는 현재 프로젝트 상태를 분석하고 다음에 어떤 스킬을 사용하면 좋을지 추천합니다.

**분석 대상:**
- `state.json` (활성 스킬, 등급, Phase)
- `current.md` (작업 상태, 차단 요소)
- 진행 중인 문서 상태
- 최근 메모리

**추천 예시:**

| 상태 | 추천 |
|------|------|
| idle (작업 없음) | `/onboarding` 또는 `/housekeeping` |
| spec 완료 | `/write` 또는 `/review` (설계 리뷰) |
| 작성 중 | `/write --continue` 또는 `/revise` |
| 리뷰 완료 | `/revise` (피드백 반영) |
| 전체 완료 | `/reflect` |
| 정리 필요 | `/housekeeping` |

추천 결과에서 번호를 말하면 바로 실행됩니다.

## 기본 흐름

### 간단한 문서: /draft

1-2페이지 분량의 간단한 메모, 이메일 등은 바로 작성합니다.

```
/draft "주간 회의 요약 메모"
```

### 본격적인 문서: /research → /spec → /write

Report 이상의 문서에서는 **조사 → 설계 → 작성** 흐름을 따릅니다.

<Steps>
### /research로 자료 수집

```
/research "Q1 매출 데이터, 채널별 성과, 경쟁사 동향"
```

`/research`는 주제를 조사하고 데이터를 수집합니다. 수집된 자료는 메모리에 저장되어 이후 `/write` 시 자동 참조됩니다.

### /spec으로 문서 구조 설계

```
/spec "Q1 마케팅 성과 보고서 — 채널별 ROI, 경쟁사 비교, Q2 전략"
```

`/spec`은 요구사항의 복잡도를 자동 분석하고 등급별 설계 문서를 생성합니다.

| 등급 | 기준 |
|------|------|
| Draft | 1-2페이지, 단일 주제 |
| Memo | 3-5페이지, 2-3개 섹션 |
| Report | 5-15페이지, 다수 섹션 + 데이터 |
| Whitepaper | 15-30페이지, 심층 분석 + 근거 |
| Publication | 30페이지+, 연작 문서 |

### /write로 본문 작성

```
/write
```

설계 문서를 기반으로 본문을 작성합니다. 작성 중 발견한 인사이트가 자동으로 메모리에 저장됩니다.

### 작업 완료 후

작성이 끝나면 회고를 진행합니다.

```
/reflect     # 작업 회고 + 인사이트 저장
```
</Steps>

## 문서 개선: /revise

작성된 문서를 개선하려면 `/revise`를 사용합니다.

```
/revise "3절 근거가 약함 — 최신 데이터 추가, 경쟁사 비교 표 삽입"
```

`/revise`는 피드백을 반영하고, 문체를 조정하고, 논리 구조를 보강합니다.

---

## 문서 리뷰: /review

문서의 품질을 검증하려면 `/review`를 사용합니다.

```
/review spec     # 설계 문서 리뷰
/review doc      # 완성된 문서 리뷰
```

Claude가 문서 주제에 맞는 전문가를 동적으로 구성하여 리뷰를 수행합니다. 결과물에 참여 전문가의 역할과 전문 분야가 명시됩니다.
Report 등급은 최대 3인, Whitepaper 이상은 최대 5인이 참여합니다.
