# /revise

작성된 문서를 **개선하고 수정**하는 스킬입니다. 리뷰 피드백 반영, 문체 통일, 논리 보강 등을 수행합니다.

---

## 언제 사용하나요?

| 상황 | 설명 |
|------|------|
| `/review` 피드백 반영 | 리뷰 결과를 문서에 적용 |
| 문체/톤 조정 | 격식체 전환, 독자 수준 변경 |
| 논리 구조 보강 | MECE 점검, 근거 보충 |
| 분량 조정 | 요약/확장 |
| 최종 교정 | 오탈자, 용어 통일, 포맷 정리 |

---

## 사용법

```
/revise                              # 최근 문서 자동 대상
/revise "피드백 또는 수정 지시"        # 구체적 수정 요청
/revise --tone formal                # 문체 변경
```

### 예시

```
/revise "3절 시장 분석 근거가 약함 — 2024 실적 데이터 추가, 경쟁사 비교 표 삽입"
```

```
/revise "전체적으로 경영진 대상 톤으로 변경, 기술 용어 최소화"
```

---

## 동작 과정

<Steps>
### 대상 문서 로드

최근 작성된 문서 또는 지정된 파일을 읽습니다.

### 수정 사항 분석

피드백, `/review` 결과, 또는 사용자 지시를 분석하여 수정 계획을 수립합니다.

### 수정 실행

섹션별로 수정을 적용합니다. 원본과 변경점을 명확히 표시합니다.

### 변경 보고

무엇이 어떻게 바뀌었는지 요약합니다.
</Steps>
