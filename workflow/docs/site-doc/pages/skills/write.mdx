# /write

spec 문서를 기반으로 **섹션별 본문을 작성**하는 스킬입니다. 문서의 실제 내용을 생성합니다.

<Callout type="warning">
  `/write`는 반드시 `/spec`으로 생성된 설계 문서가 필요합니다. 간단한 문서는 `/draft`를 사용하세요.
</Callout>

---

## 사용법

```
/write                          # 최근 spec 자동 참조
/write "spec 파일 경로"          # 특정 spec 지정
/write --section "2. 시장 분석"  # 특정 섹션만 작성
```

---

## 동작 과정

<Steps>
### Spec 로드

활성 spec 문서를 읽어 목차, 대상 독자, 핵심 논점을 파악합니다.

### 리서치 참조

`/research`로 수집된 자료가 있으면 자동으로 참조합니다.

### 섹션별 작성

목차 순서대로 섹션을 작성합니다. 각 섹션은:
- spec의 가이드라인 준수
- 근거 자료 인용 (출처 표기)
- 대상 독자 수준에 맞는 어조

### 체크포인트 저장

긴 문서는 섹션 완료마다 자동 저장합니다.

### 완료 안내

작성 완료 후 `/review`(리뷰) 또는 `/revise`(수정)를 안내합니다.
</Steps>

---

## 작성 전략

| 문서 등급 | 작성 방식 |
|----------|----------|
| Draft/Memo | 전체를 한 번에 작성 |
| Report | 섹션 단위로 나누어 작성 |
| Whitepaper+ | 챕터 단위, 중간 리뷰 포함 |

---

## 출력 위치

```
.ouroboros/docs/write/YYYY-MM-DD-제목.md
```

<Callout type="info">
  작성 중 발견된 인사이트(용어 정의, 도메인 지식 등)는 자동으로 메모리에 저장됩니다.
</Callout>
