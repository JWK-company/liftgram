# /suggest

현재 프로젝트 상태를 분석하고 **다음에 어떤 스킬을 사용하면 좋을지** 추천하는 스킬입니다.

<Callout type="info">
  무엇을 해야 할지 모르겠을 때 가장 먼저 실행하세요.
</Callout>

---

## 사용법

```
/suggest
```

별도의 인자가 필요 없습니다. 실행하면 자동으로 상태를 분석합니다.

---

## 분석 항목

| 항목 | 내용 |
|------|------|
| **활성 문서** | 진행 중인 spec, 작성 중인 문서 확인 |
| **미완료 작업** | 중단된 /write, 미반영 /review 피드백 |
| **리서치 상태** | 수집된 자료의 충분성 |
| **메모리** | 관련 인사이트, 미정리 항목 |
| **프로젝트 컨텍스트** | current.md, state.json 상태 |

---

## 추천 예시

```
현재 상태 분석 결과:

1. "Q1 마케팅 보고서" spec이 작성되어 있지만 /write가 실행되지 않았습니다.
   → /write 실행을 추천합니다.

2. 최근 /review 결과에 미반영 피드백 3건이 있습니다.
   → /revise로 피드백을 반영하세요.

3. 새로운 주제 "AI 도입 효과"에 대한 리서치가 없습니다.
   → /research "AI 도입 효과" 실행을 추천합니다.
```

---

## 활용 팁

- **세션 시작 시** 항상 `/suggest`로 시작하면 이전 작업을 이어갈 수 있습니다
- **작업 완료 후** `/suggest`로 다음 단계를 확인하세요
- **Context Compact 후** 복구 상태를 파악하는 데도 유용합니다
