# /spec

문서의 **구조를 설계**하고 **목차를 생성**하는 스킬입니다. 5-Tier 복잡도 분석을 통해 문서 규모에 맞는 설계 문서를 만듭니다.

---

## 5-Tier 복잡도 체계

| 등급 | 기준 | 산출물 |
|------|------|--------|
| **Draft** | 1-2페이지, 단일 주제 | 간략 개요 |
| **Memo** | 3-5페이지, 2-3개 섹션 | 목차 + 핵심 요점 |
| **Report** | 5-15페이지, 다수 섹션 + 데이터 | 상세 목차 + 섹션별 가이드 |
| **Whitepaper** | 15-30페이지, 심층 분석 + 근거 | 전체 설계서 + 참고자료 맵 |
| **Publication** | 30페이지+, 연작 문서 | 시리즈 설계 + 개별 문서 spec |

---

## 사용법

```
/spec "문서 설명"
```

### 예시

```
/spec "Q1 마케팅 성과 보고서 — 채널별 ROI 분석, 경쟁사 비교, Q2 전략 제안"
```

---

## 동작 과정

<Steps>
### 복잡도 분석

요청을 분석하여 Draft~Publication 등급을 자동 판정합니다.

### 메모리 조회

관련 도메인 지식, 이전 문서 패턴, 용어 정의를 검색합니다.

### 설계 문서 생성

등급에 맞는 문서 설계서를 생성합니다:
- 목적 및 대상 독자
- 목차 (섹션별 예상 분량)
- 핵심 논점 및 근거 계획
- 필요한 데이터/자료 목록

### 리뷰 안내

`/review spec`으로 설계를 검토하거나, `/write`로 바로 작성을 시작할 수 있습니다.
</Steps>

---
