# /research

문서 작성에 필요한 **주제 조사와 데이터 수집**을 수행하는 스킬입니다. 수집된 자료는 메모리에 저장되어 `/write` 시 자동 참조됩니다.

---

## 언제 사용하나요?

| 상황 | 예시 |
|------|------|
| 새로운 주제 문서 작성 전 | 시장 동향, 기술 트렌드 조사 |
| 근거 자료 확보 | 통계 데이터, 사례 수집 |
| 용어/개념 정리 | 도메인 용어 정의, 개념 비교 |
| 경쟁 분석 | 경쟁사 제품/전략 비교 |

---

## 사용법

```
/research "조사 주제"
```

### 예시

```
/research "2024년 국내 SaaS 시장 규모와 성장 전망 — B2B 중심"
```

```
/research "RAG vs Fine-tuning 비교 — 비용, 정확도, 도입 난이도 관점"
```

---

## 동작 과정

<Steps>
### 조사 계획 수립

주제를 분석하여 조사 항목, 필요 데이터 유형, 검색 전략을 수립합니다.

### 자료 수집

- **WebSearch**: 최신 자료, 통계, 보고서 검색
- **context7**: 기술 문서, API 레퍼런스 조회
- **메모리**: 기존 축적된 도메인 지식 활용

### 자료 정리

수집된 자료를 구조화합니다:
- 핵심 데이터 요약
- 출처 및 신뢰도 평가
- 인용 가능 형태로 정리

### 메모리 저장

주요 발견사항을 메모리에 저장하여 이후 `/write` 시 자동 참조되도록 합니다.
</Steps>

---

## 출력 위치

```
.ouroboros/docs/knowledge/YYYY-MM-DD-주제.md
```

  리서치 결과는 `/spec`이나 `/write` 실행 시 자동으로 참조됩니다. 별도 연결 작업이 필요 없습니다.
</Callout>
