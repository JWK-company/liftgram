# /consult

특정 분야의 **전문가에게 자문**을 구하는 경량 스킬입니다. 문서 작성 중 전문적 판단이 필요할 때 사용합니다.

<Callout type="info">
  `/review`가 완성된 문서의 종합 리뷰라면, `/consult`는 특정 질문에 대한 **포인트 자문**입니다.
</Callout>

---

## 사용법

```
/consult "질문 또는 상담 요청"
```

### 예시

```
/consult "이 재무 데이터 해석이 맞는지 확인 — 영업이익률 15%를 '양호'로 평가해도 되나요?"
```

```
/consult "법률 검토 — 이 약관 문구에 소비자보호법 위반 소지가 있는지"
```

```
/consult "이 기술 설명이 비개발자도 이해할 수 있는 수준인지 검토"
```

---

## /review와의 차이

| 항목 | /consult | /review |
|------|----------|---------|
| 범위 | 특정 질문/섹션 | 문서 전체 |
| 전문가 수 | 1-2명 | 2-5명 |
| 산출물 | 답변 + 조언 | 리뷰 보고서 |
| 소요 시간 | 짧음 | 김 |
| 파일 저장 | 선택적 | 항상 저장 |

---

## 동작 과정

<Steps>
### 질문 분석

질문의 도메인과 필요 전문성을 파악합니다.

### 전문가 구성

질문에 적합한 전문가 1-2명을 동적으로 구성합니다.

### 자문 수행

전문가 관점에서 질문에 답변하고, 관련 근거나 사례를 제시합니다.

### 결과 전달

답변 요약과 후속 조치 제안을 전달합니다. 필요 시 메모리에 저장합니다.
</Steps>

<Callout>
  자문 결과를 문서에 반영하려면 `/revise`를, 더 깊은 조사가 필요하면 `/research`를 사용하세요.
</Callout>
