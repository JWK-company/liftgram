# 기타 스킬

문서 작성의 핵심 흐름을 보조하는 **유지보수/관리 스킬**입니다.

---

## /reflect -- 회고

작업을 마친 후 **인사이트를 발견하고 메모리에 저장**합니다.

```
/reflect
```

| 저장 유형 | 설명 | 예시 |
|----------|------|------|
| `gotcha` | 예상과 다른 결과, 함정 | "이 통계는 연도별이 아니라 분기별 기준이었다" |
| `pattern` | 재사용 가능한 문서 패턴 | "경영진 보고서는 결론-근거-제안 순서가 효과적" |
| `insight` | 비자명한 발견 | "이 도메인에서는 %보다 절대값이 설득력 있다" |
| `decision` | 문서 전략 결정 | "기술 백서는 비유 중심으로 작성하기로 결정" |

<Callout>
  프로젝트 전체에 공유할 인사이트는 `scope: "project"`로 저장됩니다.
</Callout>

---

## /onboarding -- 프로젝트 동기화

프로젝트 구조와 사용자 선호를 동기화합니다.

```
/onboarding --mode project    # 프로젝트 구조 파악
/onboarding --mode user       # 사용자 문서 스타일 선호 동기화
/onboarding --mode knowledge  # 딥 분석
```

| 모드 | 용도 |
|------|------|
| `project` | 디렉토리 구조, 문서 포맷 규칙, 기존 문서 스타일 파악 |
| `user` | 선호하는 문체, 자주 쓰는 표현, 서식 습관 동기화 |
| `knowledge` | 도메인 용어, 프로젝트 배경지식 심층 분석 |

---

## /housekeeping -- 메모리 정리

축적된 메모리의 품질을 유지합니다.

```
/housekeeping
```

수행 항목:
- **중복 제거**: 유사 메모리 병합
- **Stale 검증**: 30일 이상 미확인 메모리 리뷰
- **문서 연동**: 관련 문서 변경에 따른 메모리 유효성 확인
- **Idea-Only 검사**: 원문 복사가 포함된 메모리 정리

---

## /guide -- 워크플로우 안내

Ouroboros 시스템 사용법을 안내합니다.

```
/guide              # 전체 가이드
/guide "스킬 이름"   # 특정 스킬 상세 설명
```

  이 문서 사이트와 동일한 내용을 Claude Code 내에서 바로 확인할 수 있습니다.
</Callout>
