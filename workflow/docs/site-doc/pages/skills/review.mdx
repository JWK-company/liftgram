# /review

**동적 전문가 패널**을 구성하여 문서 또는 spec을 다각도로 리뷰하는 스킬입니다.

<Callout type="info">
  전문가는 문서 주제에 따라 자동 구성됩니다. 예: 마케팅 보고서라면 "마케팅 전략가", "데이터 분석가", "카피라이터" 등.
</Callout>

---

## 사용법

```
/review spec    # 설계 문서 리뷰
/review doc     # 완성된 문서 리뷰
/review         # 가장 최근 산출물 자동 대상
```

---

## 리뷰 관점

| 관점 | 평가 항목 |
|------|----------|
| **논리 구조** | MECE 충족 여부, 논증 흐름, 결론 도출 과정 |
| **근거 품질** | 데이터 출처 신뢰도, 인용 정확성, 최신성 |
| **가독성** | 문장 명확성, 시각자료 활용, 포맷 일관성 |
| **독자 적합성** | 대상 독자 수준과 어조 일치 여부 |
| **완성도** | 누락된 섹션, 미해결 논점, TODO 항목 |

---

## 동작 과정

<Steps>
### 전문가 패널 구성

문서 주제와 복잡도에 따라 2-5명의 전문가를 자동 구성합니다.

### 병렬 리뷰

각 전문가가 자신의 관점에서 독립적으로 리뷰합니다.

### 피드백 종합

모든 피드백을 종합하여 **채택/보류/기각**으로 분류합니다.

### 리뷰 보고서 생성

우선순위별 개선 사항과 구체적 수정 제안을 포함한 보고서를 생성합니다.
</Steps>

---

## 전문가 수 기준

| 문서 등급 | 전문가 수 |
|----------|----------|
| Draft/Memo | 1-2명 |
| Report | 2-3명 |
| Whitepaper+ | 3-5명 |
