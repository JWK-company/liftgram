# Ouroboros Document Workflow

AI와 함께하는 **비즈니스 문서 작성 워크플로우**입니다. 보고서, 기획서, 백서, 메모 등 다양한 문서를 체계적으로 작성할 수 있습니다.

<Callout type="info">
  이 시스템은 **코드가 아닌 문서**를 위한 워크플로우입니다.
  작성한 문서와 리서치 결과가 서버에 축적되어, 프로젝트 전체의 문서 품질이 지속적으로 향상됩니다.
</Callout>

---

## 핵심 특징

| 특징 | 설명 |
|------|------|
| **12개 스킬** | /draft, /spec, /write, /revise, /review, /research 등 문서 작성 전 과정을 커버 |
| **5-Tier 복잡도** | 문서 규모를 자동 분석하여 Draft/Memo/Report/Whitepaper/Publication으로 분류 |
| **MCP 메모리** | 리서치 결과, 용어 정의, 문서 패턴이 서버에 축적되어 재활용 |
| **동적 전문가** | 문서 주제에 맞는 전문가 패널이 자동 구성되어 리뷰 수행 |

---

## 표준 워크플로우

```
/suggest → 현재 상태 분석, 다음 액션 추천
    ↓
/research → 주제 조사, 데이터 수집
    ↓
/spec → 문서 구조 설계, 목차 생성
    ↓
/write → 섹션별 본문 작성
    ↓
/review → 전문가 패널 리뷰
    ↓
/revise → 피드백 반영, 최종 수정
```

## 빠른 시작

<Steps>
### 설치

프로젝트 루트에서 설치 스크립트를 실행합니다.

```bash
curl -fsSL 'https://ouro.jungmin.kim/install-doc.sh?token=YOUR_TOKEN' | bash
```

설치 토큰은 [설치 페이지](https://ouro.jungmin.kim/install-doc)에서 Keycloak 로그인 후 발급받습니다.

### Claude Code 실행

```bash
claude
```

### /suggest로 시작

무엇을 해야 할지 모르겠다면 `/suggest`를 입력하세요. 현재 프로젝트 상태를 분석하고 다음 액션을 추천합니다.

### 문서 작성

간단한 문서는 `/draft`, 본격적인 문서는 `/research` → `/spec` → `/write` 순서로 진행합니다.
</Steps>

</Callout>
