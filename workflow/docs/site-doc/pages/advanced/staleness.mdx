# 메모리 최신화 관리

메모리는 시간이 지남에 따라 최신성을 잃을 수 있습니다. Ouroboros는 메모리 라이프사이클을 관리하여 항상 유효한 정보만 검색되도록 합니다.

---

## 라이프사이클

메모리는 3가지 상태를 순환합니다.

```
active → stale_candidate → archived
```

| 상태 | 검색 포함 | 설명 |
|------|----------|------|
| **active** | O | 기본 상태. 정상 사용 |
| **stale_candidate** | O (경고 표시) | 최신성 의심. 검증 필요 |
| **archived** | X | 이력 보존. 검색 제외 |

---

## stale_candidate 전환 조건

메모리가 `stale_candidate`로 전환되는 조건입니다.

### 관련 파일 변경

메모리에 연결된 `related_files`가 변경되면 해당 메모리는 더 이상 유효하지 않을 수 있습니다.

**예시:**
```
메모리: "Q1 보고서에서 시장 규모를 5조원으로 기재"
→ 관련 리서치 문서가 업데이트됨
→ stale_candidate 전환 (시장 규모 수치가 변경되었을 수 있음)
```

### 6개월 미접근

6개월 이상 검색/접근되지 않은 메모리는 자동으로 `stale_candidate`로 전환됩니다.

- 프로젝트가 변화하면서 오래된 메모리의 유효성이 감소
- 접근 빈도가 낮은 메모리는 중요도가 낮을 가능성

---

## /housekeeping으로 정리

`/housekeeping` 스킬은 stale 메모리를 체계적으로 정리합니다.

```
/housekeeping
```

**수행 작업:**

1. **중복 제거** -- 유사도가 높은 메모리 쌍을 발견하고 병합/제거
2. **Stale 검증** -- `stale_candidate` 메모리의 유효성 재확인
   - 여전히 유효하면 → `active`로 복원
   - 유효하지 않으면 → `archived`로 전환
3. **품질 보고** -- 전체 메모리 상태 통계

<Callout type="info">
  `/housekeeping`은 주기적으로 실행하는 것이 좋습니다. `/suggest`가 정리가 필요할 때 추천합니다.
</Callout>

---

## archived 메모리

`archived` 상태의 메모리는 기본 검색에서 제외됩니다.

**특징:**
- 이력 보존 목적으로 삭제하지 않음
- `--include-archived` 옵션으로 조회 가능
- 대시보드 Memories 페이지에서 `archived` 필터로 조회

**아카이브 방법:**
- `memory_archive` 도구로 수동 전환
- `/housekeeping`에서 자동 전환
- 대시보드에서 수동 전환

---

## 접근 빈도와 감쇠

메모리의 접근 빈도는 RRF 검색의 Access 축(0.15)에 반영됩니다.

```
최종 점수 = 0.4 * semantic + 0.3 * recency + 0.15 * access + 0.15 * source
```

- 자주 검색되는 메모리는 Access 점수가 높아져 상위에 노출
- 오래 접근되지 않은 메모리는 자연스럽게 순위가 하락
- 6개월 미접근 시 `stale_candidate`로 전환되어 추가 감쇠

---

## decision 타입의 버전 관리

`decision` 타입 메모리는 `superseded_by` 필드로 이력을 관리합니다.

```
decision: "보고서는 시간순 구성으로 작성"
  └─ superseded_by → decision: "피라미드 구조(결론 먼저)로 변경"
                       └─ superseded_by → decision: "경영진용은 피라미드, 실무자용은 시간순 병행"
```

- 이전 결정은 archived 되지 않고 연결 관계로 보존
- 최신 결정만 active 상태
- 결정 변경 이력을 추적 가능

<Callout>
  기술 결정은 맥락이 중요합니다. 이전 결정을 삭제하지 않고 이력으로 보존하면 "왜 이전에는 다른 선택을 했는지" 추적할 수 있습니다.
</Callout>

## 최신화 전략 요약

| 전략 | 자동/수동 | 주기 |
|------|----------|------|
| 관련 파일 변경 감지 | 자동 | 실시간 |
| 6개월 미접근 감쇠 | 자동 | 6개월 |
| /housekeeping 정리 | 수동 (권장: 주 1회) | - |
| memory_archive 수동 전환 | 수동 | - |
| 대시보드에서 관리 | 수동 | - |
