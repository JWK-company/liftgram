# 동적 전문가 시스템

Ouroboros는 고정된 에이전트 목록 대신 **동적 전문가 시스템**을 사용합니다. Claude가 문서 주제를 분석하여 해당 리뷰에 최적화된 전문가 역할을 자동으로 정의하고 호출합니다.

---

## 동작 원리

### 1. 문서 주제 분석

`/review`가 호출되면 Claude는 다음을 분석합니다:

- 문서 유형 (보고서, 백서, 기획서, 제안서 등)
- 문서 등급 (Draft / Memo / Report / Whitepaper / Publication)
- 현재 문서의 핵심 주제와 대상 독자

### 2. 전문가 역할 자동 정의

분석 결과를 바탕으로 해당 리뷰에 필요한 전문가의 역할과 전문 분야를 즉석에서 구성합니다. 미리 정해진 역할 목록이 아니라, 문서에 가장 적합한 관점의 조합이 선택됩니다.

### 3. 전문가 호출 및 리뷰

구성된 전문가 역할로 Claude가 각 관점에서 순차적으로 리뷰를 수행합니다.

### 4. 결과 명시

리뷰 결과에는 참여한 전문가의 역할과 전문 분야가 명시되어, 어떤 관점에서 검토가 이루어졌는지 확인할 수 있습니다.

---

## 등급별 전문가 인원

| 등급 | 참여 인원 | 설명 |
|------|----------|------|
| **Memo/Draft** | 1-2인 | 핵심 관점만 압축 |
| **Report** | 2-3인 | 주요 관심사 커버 |
| **Whitepaper / Publication** | 3-5인 | 다각도 심층 리뷰 |

<Callout type="info">
  Draft/Memo 등급에서도 `/review`로 명시적 호출이 가능합니다. 이 경우 문서 규모에 맞게 1-2인의 전문가가 구성됩니다.
</Callout>

---

## 구성 예시

### 마케팅 보고서 리뷰

```
/review doc   # Q1 마케팅 성과 보고서 리뷰
```

구성 예시:
- **마케팅 전략가** -- 채널별 ROI 분석의 적절성, 경쟁사 비교 방법론, 전략 제안의 실현 가능성
- **데이터 분석가** -- 통계 해석 정확성, 데이터 출처 신뢰도, 시각화 적절성
- **카피라이터** -- 문장 명확성, 경영진 대상 어조 적합성, 핵심 메시지 전달력

### 기술 백서 리뷰

```
/review doc   # AI 도입 전략 백서 리뷰
```

구성 예시:
- **기술 전략가** -- 기술 비교 객관성, 도입 로드맵 현실성, 리스크 분석 충분성
- **재무 분석가** -- ROI 산출 방법론 적절성, 비용 추정 정확성, 투자 대비 효과 분석
- **도메인 전문가** -- 업종별 사례 적합성, 도메인 용어 정확성, 실무 적용 가능성

### 제안서 리뷰

```
/review doc   # 신규 서비스 제안서 리뷰
```

구성 예시:
- **사업 기획 전문가** -- 시장 분석 타당성, 경쟁 우위 논리, 사업 모델 완성도
- **고객 관점 평가자** -- 고객 가치 제안 명확성, 사용자 시나리오 현실성
- **재무 감사관** -- 수익 모델 논리성, 손익 추정 합리성, 투자 회수 기간 산정

## 리뷰 결과 형식

리뷰가 완료되면 각 전문가의 의견이 종합되어 다음 형태로 보고됩니다:

```
[마케팅 전략가] 리뷰 결과
  Critical: 경쟁사 비교에서 시장 점유율 데이터 출처가 누락됨
  Warning: Q2 전략 제안이 Q1 분석 결과와 연결이 약함

[데이터 분석가] 리뷰 결과
  Critical: 3절의 전환율 계산 방식에 오류 가능성 (분모 기준 불일치)
  Suggestion: 채널별 성과를 시계열 차트로 시각화하면 트렌드 파악이 용이

[카피라이터] 리뷰 결과
  Warning: 경영진 대상이면 기술 용어를 비즈니스 임팩트로 번역할 필요
```

- **Critical** -- 반드시 수정해야 하는 사항
- **Warning** -- 수정을 권장하는 사항
- **Suggestion** -- 개선 제안

<Callout type="warning">
  Critical 항목이 있으면 최종 제출 전에 반드시 해결하세요.
</Callout>
