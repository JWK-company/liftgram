# ADR-011 · 백엔드 토폴로지 = 모듈러 모놀리스 (마이크로서비스 지양)

## 맥락 (Context)
백엔드는 인증·동기·소셜·AI 오케스트레이션·결제 검증·콘텐츠를 도메인으로 다룬다(SAD-001). 부트스트랩 소팀이며 초기 트래픽 규모가 불확실하다. 마이크로서비스는 운영·배포·관측 오버헤드가 크고, 조기 분산은 소팀에 독이다.

## 결정 (Decision)
백엔드를 **모듈러 모놀리스**(도메인별 경계가 명확한 단일 배포 단위)로 시작한다. 도메인 간 경계는 모듈로 분리해 추후 특정 도메인(예: AI 추론·동기)이 독립 확장 필요 시 분리할 수 있도록 한다.

## 근거 / 대안
- 채택 이유: 소팀 운영 단순성, 빠른 반복, 낮은 인프라 비용. 모듈 경계 유지로 후기 분리 옵션 보존.
- 검토한 대안: ① **마이크로서비스** — 독립 확장·격리 우수하나 운영·관측·배포 복잡도가 소팀엔 과도. ② **서버리스 함수 우선** — 일부 워크로드(AI)엔 적합하나 전면 적용은 상태·로컬개발 복잡.

## 결과 (Consequences)
- 장점: 단순 배포·디버깅·낮은 비용, 빠른 출시.
- 트레이드오프: 단일 장애 도메인 격리 약함, 특정 도메인 독립 확장 시 분리 작업 필요.
- 되돌림 비용: 중 — 모듈 경계를 지키면 도메인 추출이 비교적 수월(경계 무너지면 비용↑).

## 연결
- 영향 SRS/SAD: SAD-001(시스템 개요·토폴로지) · 대체: 없음
