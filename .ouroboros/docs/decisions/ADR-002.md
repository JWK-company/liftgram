# ADR-002 · 로컬-우선(offline-first) 동기화와 엔티티별 충돌 해소 모델

## 맥락 (Context)
운동은 헬스장 지하·비행기 모드 등 네트워크가 불안정한 환경에서 기록된다. 기록이 끊기거나 손실되면 코어 신뢰가 무너진다(SRS-006). 다기기 사용 시 동시 편집 충돌도 발생한다. 따라서 네트워크 의존 없이 즉시 쓰고, 복귀 시 안전하게 병합하는 모델이 필요하다.

## 결정 (Decision)
**로컬-우선**을 1차 진실 소스로 삼고 백그라운드 양방향 동기화한다. 충돌은 **엔티티 유형별 정책**으로 해소한다: 세트/세션 로그 = **append-only(추가 전용, 무손실)**, 메타데이터(루틴 이름 등) = **버전 벡터 기반 last-writer-wins**.

## 근거 / 대안
- 채택 이유: 운동 로그는 본질상 추가형이라 append-only가 자연스럽고 손실이 없다. 메타는 단순 LWW로 충분. 구현 복잡도와 안전성의 균형.
- 검토한 대안: ① **풀 CRDT** — 강력하나 전 엔티티 적용은 과설계·복잡도 폭증. ② **서버-우선(온라인 전제)** — 오프라인 기록 불가 → 코어 요구 위반. ③ **단순 LWW 전체** — 로그 충돌 시 데이터 손실 위험.

## 결과 (Consequences)
- 장점: 오프라인 완전 동작, 무손실 로그, 예측 가능한 병합.
- 트레이드오프: 동기 엔진·버전 벡터 관리 복잡도, 클라이언트 저장소 설계 부담.
- 되돌림 비용: 중상 — 동기 모델 변경은 데이터 마이그레이션 수반.

## 연결
