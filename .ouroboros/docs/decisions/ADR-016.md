# ADR-016 · 미디어 저장·CDN·트랜스코딩 스택 — 오브젝트 스토리지 + CDN + 매니지드 트랜스코딩

## 맥락 (Context)
SRS-019(사진/영상 게시물·24h 스토리)는 업로드·저장·CDN 전송·영상 트랜스코딩·썸네일·만료를 요한다. 미디어는 대용량·고대역폭이라 직접 운영(스토리지+자체 CDN+ffmpeg 파이프라인)은 비용·복잡도·장애 리스크가 크다. 부트스트랩 단계에서 전송 성능과 운영부담의 균형이 필요하다.

## 결정 (Decision)
**S3 호환 오브젝트 스토리지(예: Cloudflare R2·AWS S3) + CDN(예: Cloudflare) + 매니지드 영상 트랜스코딩(예: Cloudflare Stream·Mux)** 을 채택한다. 클라이언트는 **서명 URL로 직접 업로드**하고, 파생물(트랜스코딩·썸네일)은 파이프라인이 생성하며, 공개범위는 서명·접근제어로 반영한다. 스토리 자산은 **24h TTL**로 만료·정리한다.

## 근거 / 대안
- 채택 이유: 확장·대역폭·트랜스코딩을 매니지드로 위임해 소팀 운영부담을 최소화하고 전송 성능을 확보. R2는 egress 무료로 미디어 전송 비용에 유리.
- 검토한 대안: ① **자체 ffmpeg + 자체 CDN** — 통제·단가는 좋으나 운영 난도·장애 리스크 과대. ② **외부 영상 호스팅 임베드(유튜브 등)** — 통제·정책·UX 제약, 소셜 일체감 저해.

## 결과 (Consequences)
- 장점: 빠른 확장·낮은 운영부담·egress 비용 유리(R2).
- 트레이드오프: 벤더 종속·트랜스코딩 단가.
- 되돌림 비용: 중간 — S3 호환 표준이라 스토리지 이전은 용이하나, 트랜스코딩 교체는 재인코딩 필요.
