# ADR-015 · 실시간 메시징 백엔드·전송 기술 — 매니지드 챗 우선 + 추상화 레이어

## 맥락 (Context)
SRS-017(DM — 1:1·그룹·실시간 전송·읽음 상태·푸시·신고/차단)을 백엔드 0%에서 구축해야 한다. 실시간 메시징은 전송 채널(WebSocket)·메시지 영속·순서·오프라인 동기·푸시·모더레이션 훅까지 난도가 높고, 부트스트랩 소팀이 자체 구현하면 구축·운영 비용·장애 리스크가 크다.

## 결정 (Decision)
초기에는 **매니지드 실시간 챗 백엔드**(예: Stream Chat·Sendbird·Supabase Realtime 등급)를 채택한다. 단, 앱 측은 **메시징 추상화 레이어**로 감싸 후일 자체호스팅/벤더 교체가 가능하게 한다. 전송은 WebSocket 기반, 메시지 영속·읽음 상태·채널(1:1/그룹)·푸시·모더레이션 훅은 매니지드 기능을 활용한다. 음성/영상 통화는 본 결정 범위 밖(SRS-017과 동일하게 후속).

## 근거 / 대안
- 채택 이유: 실시간성·확장·푸시·안전(신고/차단/모더레이션 훅)을 빠르게 확보하고 소팀 운영부담을 최소화. 출시 속도 우선.
- 검토한 대안: ① **자체호스팅 WebSocket + 메시지 큐(Kafka 등)** — 통제·단가는 좋으나 초기 구축/운영 난도·장애 리스크 과대. ② **폴링 기반** — 실시간성 부족·비효율. ③ **벤더 종속 우려** — 추상화 레이어로 교체 비용을 완화.

## 결과 (Consequences)
- 장점: 빠른 출시·확장·안전 기능 내장.
- 트레이드오프: 벤더 종속·메시지 단가, 커스텀 제약.
- 되돌림 비용: 중간 — 추상화로 완화하나 메시지 데이터 마이그레이션은 필요.

## 연결
- 영향 SRS/SAD: SAD-011(소셜·메시징 백엔드), SRS-017(DM) · 대체: 없음
