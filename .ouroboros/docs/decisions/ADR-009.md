# ADR-009 · 구독 인프라는 서드파티(RevenueCat) 채택

## 맥락 (Context)
구독(SRS-014)은 iOS/Android 두 스토어의 인앱결제·영수증 검증·갱신·해지·복원·환불·구독 상태 동기화를 다뤄야 한다. 이를 직접 구축하면 양 플랫폼 결제 엣지케이스 대응에 상당한 공수가 들고, 소팀의 핵심 차별화(코칭·책임감)에서 자원을 뺏긴다.

## 결정 (Decision)
구독 인프라로 **서드파티(RevenueCat)**를 채택하여 인앱결제·엔타이틀먼트·크로스플랫폼 구독 상태를 위임한다. 서버 엔타이틀먼트 서비스(SAD-009)는 RevenueCat 웹훅/SDK로 권한을 동기화한다.

## 근거 / 대안
- 채택 이유: 양 스토어 결제 복잡도를 검증된 SaaS로 흡수 → 출시 가속, 핵심 기능에 집중. RN SDK 지원 양호.
- 검토한 대안: ① **자체 구축** — 완전 제어·수수료 절감이나 결제 엣지케이스·유지보수 부담 큼. ② **스토어 SDK 직접 연동** — 중간이나 크로스플랫폼 상태 통합·검증 직접 부담.

## 결과 (Consequences)
- 장점: 빠른 구현, 결제 신뢰성, 분석·실험 기능 제공.
- 트레이드오프: 매출 연동 수수료·외부 의존성, 데이터가 서드파티 경유.
- 되돌림 비용: 중 — 엔타이틀먼트 추상화 시 후기 자체 전환 가능하나 마이그레이션 필요.
