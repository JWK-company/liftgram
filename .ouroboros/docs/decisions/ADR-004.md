# ADR-004 · AI 프로그래밍은 규칙엔진·템플릿 우선 + LLM 보조 하이브리드

## 맥락 (Context)
차별화 핵심인 AI 프로그래밍(SRS-009/010)은 점진적 과부하·주기화라는 검증된 운동과학 원칙을 따라야 하고, 부상·과대표현 리스크가 있어 출력이 안전·설명가능·재현가능해야 한다(SAD-006·SAD-010). 순수 LLM 자유 생성은 환각·일관성·안전 통제가 어렵다.

## 결정 (Decision)
프로그램 생성·주차 진행을 **결정적 규칙 엔진 + 운동과학 템플릿을 1차**로 하고, **LLM은 보조**(자연어 설명, 종목 다양화 제안, 사용자 맥락 해석)로 제한한다. LLM 사용 시 **최신 Claude 모델**을 기본으로 하고, 출력은 가드레일(ADR-006)을 통과한 뒤 노출한다.

## 근거 / 대안
- 채택 이유: 안전·설명가능·재현성을 규칙으로 보장하면서, LLM으로 UX(설명·개인화)를 강화. 점진 과부하·디로드 같은 핵심 로직은 규칙으로 검증 가능.
- 검토한 대안: ① **순수 LLM 생성** — 유연하나 환각·비일관·안전 통제 난이도↑, 의료/부상 리스크. ② **순수 규칙엔진** — 안전하나 경직·개인화·자연어 설명 약함. → 하이브리드가 균형.

## 결과 (Consequences)
- 장점: 안전·재현성과 개인화 UX의 균형. 핵심 로직 테스트 용이.
- 트레이드오프: 규칙 엔진과 LLM 경계·오케스트레이션 복잡도, 운동과학 규칙 큐레이션 부담.
- 되돌림 비용: 중 — 생성 계층이 모듈화되면 비중 조정 가능.

## 연결
- 영향 SRS/SAD: SAD-006(AI 프로그래밍), SRS-009·SRS-010 · 대체: 없음
