# SAD-013 · 결제·정산 아키텍처 (웹 MoR 수금 · User-centric 구독풀 배분 · 글로벌 Payout)

## 개요 (Summary)
ADR-013(플랫폼 Merchant of Record 수금 + 유튜브식 구독풀 User-centric 트레이너 배분 + 글로벌 Payout)을 구현하는 양면 마켓플레이스 결제·정산 아키텍처다. **수금 측(SRS-014)** 은 웹 MoR 전문사(Paddle/Lemon Squeezy)에 위임해 플랫폼이 판매자가 되고 전세계 소비세를 자동 징수·납부하며 앱스토어 IAP(15~30%)를 회피한다. **정산 측(SRS-016)** 은 서버에서 구독풀을 구독자별 사용량 기반(User-centric)으로 트레이너에게 배분하고, 글로벌 대량지급으로 통화·원천징수·세무서류를 처리한다. 앱(클라이언트)은 결제 진입·구독 관리·엔타이틀먼트 표시·정산 대시보드 조회만 담당하고, 금전 계산·지급은 전부 서버 권위.

## 컴포넌트 / 책임
- **MoR Checkout & Subscription**: 웹 MoR(Paddle/Lemon Squeezy) 연동 — 구독 수금·생명주기(해지/복원)·전세계 소비세 위임·웹훅 수신/검증(SRS-014).
- **Entitlement Service**: 구독 상태 → 프리미엄 권한 매핑·기능 게이팅·클라이언트 동기(SRS-014).
- **Engagement Metering**: 정산기간 내 구독자별 트레이너 사용량(해당 트레이너 루틴 기반 세션 수행/완주 — SRS-003/004/005가 원천) 집계 → User-centric 배분 입력(SRS-016).
- **Revenue Allocation Engine**: 구독자별 net(구독료 − MoR 수수료 − 플랫폼 take rate)을 그 구독자가 실제 사용한 트레이너에게 인게이지먼트 비중으로 배분(미사용 0), 셀프구독·가짜완주 어뷰징 탐지·차단(SRS-016).
- **Global Payout Engine**: 대량지급 사업자(Trolley/Tipalti) 연동 — 수령인 국가별 통화 지급·미국 원천징수(조세조약율)·세무서류(1099/1042-S·EU DAC7)·임계치 미만 이월(SRS-016).
- **Ledger & Reconciliation**: 거래·배분·지급 원장(double-entry)·대사·감사 추적(분쟁·환불 정합).
- **Trainer Settlement Dashboard API**: 예상/확정 수익·산정 근거·지급 상태 제공(SRS-016).

## 주요 결정 포인트 (→ ADR 후보)
- 수금/배분/지급 모델(MoR + User-centric + 글로벌 Payout)은 ADR-013 결정.
- 결제 PII·세무 민감정보 보호·동의는 ADR-017(개인정보·안전)·SAD-010(웰니스/컴플라이언스) 연계.
- 어뷰징 방지(셀프구독·가짜완주) 탐지 규칙 상세 → 후속 ADR/정책 후보.

## 연결
- 다루는 SRS: SRS-014(웹 MoR 수금), SRS-016(트레이너 수익배분·정산·지급)
- 상위 결정: ADR-013(글로벌 결제·정산 모델) · 컴플라이언스: SAD-010 · 데이터 원천: SRS-003/004/005(인게이지먼트)
