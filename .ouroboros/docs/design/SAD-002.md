# SAD-002 · 운동 기록 코어 (운동DB·루틴·세션·세트 로깅)

## 개요 (Summary)
운동 기록 코어는 제품의 심장으로, **운동 카탈로그 → 루틴 → 라이브 세션 → 세트 로그**로 이어지는 데이터·상태 흐름을 담당한다. 모든 입력은 로컬-우선으로 즉시 저장되어 마찰(≤3탭)과 복구 안정성을 보장하며, 세션 종료 시 분석 엔진(SAD-003)으로 이벤트를 흘려보낸다. 이 코어가 Hevy 수준의 "가장 단순한 로깅" 기준선을 충족시킨다.

## 컴포넌트 / 책임
- **Exercise Catalog**: 종목 속성(근육군·기구·유형·한/영 별칭)과 대체운동 그래프 보관·검색·필터. 커스텀 종목 등록 지원(SRS-001).
- **Routine Builder**: 루틴 CRUD·정렬·슈퍼셋 묶기·폴더/복제. 루틴은 세션 템플릿으로 인스턴스화(SRS-002).
- **Session Manager**: 세션 생애주기(시작/진행/일시정지/종료), 루틴 기반·빈 운동 시작, 진행 상태 영속화·복구, 종료 요약(볼륨·시간·PR 플래그) 생성(SRS-004).
- **Set Logger**: 세트별 무게/횟수/RPE 입력, 직전 기록 자동표시·자동채움, 휴식 타이머, 단위(kg/lb)·플레이트 계산(SRS-003).
- **Local Store(요약)**: 위 엔티티의 로컬-우선 영속화(상세 동기 SAD-004).

## 주요 결정 포인트 (→ ADR 후보)
- 운동 DB 시드 출처/구축 방식(자체 큐레이션 vs 오픈데이터) → ADR.
- 로컬 영속화 엔진 선택(WatermelonDB/SQLite/Realm 등) → ADR.
