# SAD-011 · 소셜·실시간 메시징 백엔드 (소셜그래프·피드/디스커버리·DM·알림 fan-out)

## 개요 (Summary)
현재 앱은 완전 로컬(WatermelonDB·동기 0%)이므로 소셜 고도화는 **서버 백엔드를 전제**로 한다. 본 설계는 계정·동기(SRS-006·SAD-004) 위에 **소셜 그래프·피드/디스커버리·실시간 다이렉트 메시징·통합 알림**을 하나의 백엔드 도메인으로 묶는다. 포지셔닝은 '좁은 부족' 밀도(ADR-007)를 1급으로 유지하면서 **옵트인 공개 레이어**(ADR-014)를 노출·랭킹 정책으로 얹는 하이브리드다. 실시간 메시징의 전송·저장 방식은 ADR-015 결정에 따르고, 미디어는 SAD-012가 담당하며 본 백엔드는 미디어 참조(ID/URL)만 다룬다.

## 컴포넌트 / 책임
- **Identity & Sync Gateway**: 인증·세션·기기 간 동기(SRS-006, SAD-004 연계). 소셜 데이터의 서버측 SSOT·로컬 캐시 동기.
- **Social Graph Service**: 팔로우/팔로워·공개범위(전체/팔로워/비공개)·차단 관계·멘션 대상 해석(SRS-008/018).
- **Feed & Discovery Service**: 팔로잉·그룹 기반 시간순 피드 + 미디어 포스트 합성, 탐색/검색·추천 랭킹(부족 우선·글로벌 보조), 해시태그 인덱스(SRS-007/018).
- **Direct Messaging Service**: 1:1·그룹 대화방·메시지 순서/전달·전송/읽음 상태·수신범위/요청 메시지·차단(SRS-017). 실시간 전송 채널(ADR-015).
- **Notification Fan-out**: 좋아요·댓글·팔로우·DM·멘션·케어 이벤트 집계 → 인앱+푸시 fan-out, 유형별 on/off·다발 집계(SRS-020). 책임감 케어(SRS-011)와 톤 일관.
- **Safety & Report Queue**: 신고·차단 처리 큐 — 모더레이션(SAD-012)·정책(ADR-017)과 연동.

## 주요 결정 포인트 (→ ADR 후보)
- 실시간 메시징 전송·저장 기술(매니지드 챗 SDK vs 자체호스팅 WebSocket) → ADR-015.
- 포지셔닝: '좁은 부족' 우선 + 옵트인 공개 레이어 노출/랭킹 정책 → ADR-014(ADR-007 확장).
- 안전·모더레이션·민감정보(건강·미성년) 정책 연동 → ADR-017.

## 연결

- 미디어 저장·전송·모더레이션: SAD-012
