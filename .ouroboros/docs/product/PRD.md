# PRD · 헬스앱 (health-practice)

> 한 줄 정의: **"내 운동을 가장 빠르게 기록하고, AI가 짜주고, 동료와 함께 지키는 근력운동 앱."**
> 전략 근거: `reference/` (health-app.md · hevy-success-analysis.md · risks.md · 20260331_workout_excel_builder.md)
> 포지셔닝 결정(2026-06-23): **코어 클론 → 차별화 레이어** · 플랫폼 **React Native 크로스플랫폼** · wedge **A·E·B + 책임감 소셜 루프 전부 채택**.

## 문제

1. **순수 운동 기록은 이미 상품화(commodity)됨** — Hevy(1,400만 유저)·Strong·JEFIT가 무료로 잘 한다. 정면 클론만으로는 차별화 0 = 죽음(reference/hevy-success-analysis §4).
2. **그러나 셋 다 못 푸는 빈칸이 있다**:
   - Hevy = 로거(기록·소셜)뿐, **프로그램을 짜주지 않음**("hands-off, 도구만 준다" 공식 철학).
   - Fitbod = 적응형 코칭은 있으나 **소셜이 없고 비쌈**(글로벌·월 $15.99).
   - Planfit = AI 플래너+한국 로컬이나 **능동적 책임감(빠지면 끌어주는 구조)이 없음**.
3. **구조적 이탈** — 헬스앱 공통 약점 **D30 리텐션 3~10%**. 코칭("뭘 할지")은 풀어도 **"왜 계속하는지"(능동적 책임감)는 누구도 못 품**.
4. **로깅 마찰** — 모든 앱이 수동 탭 입력. 세트마다 무게·횟수를 손으로 치는 마찰이 남아 있다.
5. **한국 환경 부적합** — 글로벌 앱은 한국 헬스장 기구·한국어 운동 용어·오운완 인증 문화에 약하다.

## 목표 / 비목표

### 목표
- **G1 (코어)**: Hevy 수준의 빠르고 직관적인 근력운동 로깅·루틴·분석·소셜 코어를 RN 크로스플랫폼으로 완성. "세상에서 가장 단순한 운동 기록"의 진입장벽 0(무료·무광고).
- **G2 (차별화 — Hevy가 안 하는 곳)**:
  - **Wedge A**: AI 지능형 프로그래밍/주기화 — 목표·경력·장비·일정 입력 → 점진적 과부하 프로그램 자동 생성 + 주차별 자동 진행/디로드.
  - **책임감 소셜 루프**: 운동 메이트·그룹 챌린지·결석 케어 — D30 리텐션 엔진이자 성장(추천·그룹 바이럴) 엔진.
  - **Wedge E**: 센서/카메라 기반 자동 횟수 카운팅 — 로깅 마찰 제거.
  - **Wedge B**: 한국 로컬라이즈 — 헬스장 기구 기반 루틴·한국어 운동 DB·오운완 인증 문화.
- **G3 (성장·수익)**: SEO 콘텐츠 엔진(Hevy 트래픽 77% 검색 유입 모델) + 무료 진입 → 프리미엄(AI코치 심화) + 그룹/챌린지 수익화. 부트스트랩 가능한 구조.

### 비목표 (의도적으로 안 함)
- **의료기기 영역**: 진단·치료·예방·"질병 개선/살 빠진다" 표방 금지 → **웰니스 포지션 유지**(디지털의료제품법·의료기기법 식약처벽 회피). reference/health-app §법적.
- **범용 만능앱**: 칼로리 식단·러닝·홈트 풀세트 만들지 않음. **근력운동(weight/strength) 니치**에 집중(Hevy 교훈 ⑥).
- **하드 의료 데이터 통합 1차 제외**: HealthKit/Health Connect 심화 연동은 후순위(코어 안정화 후).
- **엑셀 빌더 정면 제품화 금지**: 엑셀 import는 온보딩 다리로만(reference의 HOLD 4.2 교훈). 단독 가치 아님.

## 페르소나

- **P1 — 중급 리프터 "기록러"** (코어 타깃): 이미 운동 기록 습관이 있고 빠른 로깅·진척 그래프·PR 추적·직전기록 자동표시를 원한다. Hevy/Strong 사용자와 겹침.
- **P2 — 초·중급 "프로그램이 필요한 사람"** (Wedge A 타깃): 뭘 어떻게 해야 할지 몰라 정체. AI가 짜주는 점진적 과부하 프로그램과 주차 진행이 필요.
- **P3 — "혼자선 못 버티는 사람"** (책임감 루프 타깃): 의지가 약해 자주 이탈. 같은 목표의 동료·그룹 챌린지·결석 케어가 있어야 지속.
- **P4 — 한국 헬스장 이용자** (Wedge B 타깃): 한국어 운동명·동네 헬스장 기구 기반 루틴·오운완 인증 문화에 익숙.

## 범위

### Phase 0 — 코어 클론 (P0, MVP 토대)
운동 DB(근육군·기구·대체운동) · 루틴 빌더 · 세트 로깅(무게/횟수/RPE, 직전기록 자동표시, 휴식 타이머) · 운동 세션(슈퍼셋·진행중 저장) · 분석 대시보드(볼륨·1RM 추정·PR·진척 그래프) · 계정/인증 + 로컬 우선 + 크로스 디바이스 싱크.

### Phase 1 — 소셜 코어 (P1)
팔로우 · 운동 피드 · 좋아요/댓글 · 프로필 · PR 공유. (소셜은 "넓게"면 죽음 → 부족/그룹 밀도 중심 설계.)

### Phase 2 — 차별화 레이어 (P1~P2)
- **AI 지능형 프로그래밍/주기화** (Wedge A)
- **능동적 책임감 소셜 루프** (운동 메이트·그룹 챌린지·결석 케어)
- **자동 횟수 카운팅** (Wedge E)
- **한국 로컬라이즈** (Wedge B)

### 횡단 (Cross-cutting)
- **수익화**: 무료 티어 + 프리미엄(AI코치 심화·고급 분석) + 그룹/챌린지·PT 마켓.
- **웰니스/컴플라이언스 가드레일**: 의료기기벽 회피 카피·민감정보(건강) 동의·처리.
- **성장**: SEO 콘텐츠 엔진(운동 가이드·검색 유입).

## 성공 지표

| 영역 | 지표 | 목표(가설) |
|------|------|-----------|
| 핵심 활성 | 세트 로깅 마찰 | 세트 1건 기록 ≤ 3탭, 자동 카운팅 시 0탭 |
| 리텐션 | **D30 리텐션** | 업계 3~10% → **그룹 가입 코호트 ≥ 25%** (책임감 루프 가설) |
| 차별화 채택 | AI 프로그램 생성 후 4주 완주율 | ≥ 40% |
| 소셜 밀도 | 그룹당 주간 활성 멤버 | ≥ 60% (좁은 부족 = 꽉 찬 느낌) |
| 성장 | SEO 유입 비중 | 신규 유저의 ≥ 50% (Hevy 77% 모델 지향) |
| 수익 | 무료→프리미엄 전환 | 업계 중앙값 2.18% 초과 (그룹 가치 묶기로) |

## 핵심 차별화 가설 (플라이휠)

```
좁은 부족(tribe) 진입
  → AI 코칭이 "오늘 뭘 할지" 마찰 제거 (Wedge A)
  → 자동 카운팅이 "기록 마찰" 제거 (Wedge E)
  → 같은 부족 메이트·챌린지 = 빠지면 끌어줌 (책임감 루프)
  → 리텐션↑ → 부족 내 소셜 밀도↑ → 추천으로 새 멤버
  → SEO 콘텐츠로 부족 유입 (저비용 획득) → 다시 처음으로
```

## 리스크 (요약 — 상세는 ADR/리서치)
- **법규**: 식약처 의료기기벽 — 카피·기능을 웰니스로 한정. AI 코칭/카운팅 부상 시 제조물책임 → 면책·안전 가드레일 필요.
- **경쟁**: Planfit이 Wedge A·B를 상당 부분 점유 → 단독으론 정면충돌. **책임감 루프 + 좁은 부족 밀도**가 진짜 해자.
- **기술**: 자동 카운팅(MediaPipe/센서)=중상 난이도·정확도·종목 커버 부담 → 좁게 시작.
- **IP**: 본업 회사 IP 귀속(발명양도·자산 분리) — reference/risks.md 착수 전 점검.
