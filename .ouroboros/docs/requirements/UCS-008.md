# UCS-008 · 동의를 관리하고 내 건강 데이터 열람·삭제를 요청한다

## 액터
자신의 건강 민감정보를 통제하려는 사용자. 부차 액터: 동의/프라이버시 서비스(SAD-010).

## 사전 조건
- 로그인되어 있고 최초 가입 시 목적별 동의 화면을 거쳤다.

## 주 흐름
1. 사용자가 설정 > 개인정보/동의를 연다.
2. 목적별 동의 현황(건강 데이터 수집·카운팅 영상 처리·AI 코칭 등)을 확인한다.
3. 사용자가 특정 동의를 철회하거나 다시 허용한다.
4. 사용자가 "내 데이터 열람/내보내기" 또는 "데이터 삭제"를 요청한다.
5. 시스템이 요청을 접수·처리하고 결과(내보내기 파일 제공 또는 삭제 완료)를 통지한다.

## 대안 / 예외 흐름
- 3a. 특정 기능(예: 자동 카운팅)에 필수인 동의를 철회하면 해당 기능이 비활성화됨을 고지하고 흐름을 막지 않는다.
- 4a. 삭제 요청 시 되돌릴 수 없음을 확인받고, 법정 보관 의무가 있는 항목은 범위를 명시한다.
- 5a. 처리 지연 시 접수 상태와 예상 처리 기한을 안내한다.

## 사후 조건
- 동의 상태 변경·열람/삭제 요청 이력이 감사 가능하게 기록된다.
- 삭제 완료 시 해당 민감정보가 보존 예외를 제외하고 제거된다.

## 연결
- 구체화 대상 URS: URS-008(안전·신뢰/웰니스)
