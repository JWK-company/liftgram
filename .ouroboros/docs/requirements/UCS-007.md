# UCS-007 · 온보딩하고 기존 운동 기록을 가져온다

## 액터
앱을 처음 쓰는 신규 사용자(P1·P4). 부차 액터: 외부 시트/엑셀, 타 앱 내보내기 파일.

## 사전 조건
- 앱이 설치되어 있고 네트워크에 연결되어 있다.

## 주 흐름
1. 사용자가 가입(이메일/소셜)하고 기본 프로필(목표·경력·선호 언어=한국어)을 설정한다.
2. 사용자가 다니는 헬스장의 가용 기구를 선택한다(한국 로컬, 선택).
3. 앱이 "기존 기록 가져오기"를 제안한다 — 엑셀/구글시트 또는 타 앱 내보내기 파일 업로드.
4. 업로드된 데이터의 컬럼(종목·무게·횟수·날짜)을 매핑하고 미리보기로 확인한다.
5. 사용자가 확정하면 기록이 앱 데이터로 변환·저장되고 분석에 반영된다.
6. 온보딩 완료 후 첫 루틴 생성 또는 AI 프로그램 생성으로 안내된다.

## 대안 / 예외 흐름
- 3a. 가져올 기록이 없으면 마이그레이션을 건너뛰고 빈 상태로 시작한다.
- 4a. 컬럼 매핑이 모호하면 자동 추정 후 사용자가 수정한다.
- 5a. 일부 행이 파싱 실패하면 실패 항목만 리포트하고 나머지는 정상 반영한다.

## 사후 조건
- 사용자 계정·프로필·(선택)가용 기구·(선택)가져온 기록이 생성되어 이후 흐름의 토대가 된다.
- 마이그레이션은 온보딩 다리로만 작동하며 단독 제품화하지 않는다(reference 교훈).

## 연결
- 구체화 대상 URS: URS-001(빠른 로깅), URS-007(한국 환경)
