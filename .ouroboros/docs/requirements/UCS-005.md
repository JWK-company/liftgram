# UCS-005 · 그룹 챌린지에 참여하고 결석 케어를 주고받는다

## 액터
지속이 어려운 사용자(P3)와 같은 그룹 동료들.

## 사전 조건
- 로그인되어 있고 운동 메이트/그룹 매칭 또는 챌린지가 존재한다.
- 케어 알림 수신/발신에 대한 사용자 설정이 지정되어 있다.

## 주 흐름
1. 사용자가 같은 목표의 그룹/챌린지를 찾거나 운동 메이트로 매칭된다.
2. 챌린지 규칙(기간·주당 목표 횟수 등)을 확인하고 참여한다.
3. 멤버들의 수행 현황이 그룹 보드에 가시적으로 공유된다.
4. 시스템이 사용자의 계획 운동일 미수행을 감지하면 본인에게 넛지를 보낸다.
5. 지속 미수행 시(설정에 따라) 동료에게 케어 신호가 전달되고, 동료가 응원/체크인 메시지를 보낸다.
6. 챌린지 종료 시 달성 결과가 집계되어 그룹에 공유된다.

## 대안 / 예외 흐름
- 4a. 사용자가 휴식일/부상을 사전 등록하면 결석으로 집계하지 않는다.
- 5a. 사용자가 케어 신호 공유를 비활성화하면 동료 알림 없이 본인 넛지만 발생한다(강제·수치심 방지).
- 5b. 반복 무응답 시 압박을 높이지 않고 지지형 메시지로 톤을 유지한다.

## 사후 조건
- 그룹/챌린지 진행 상태·케어 이벤트가 기록되고 리텐션 지표 분석에 활용된다.
- 케어 상호작용이 소셜 그래프 밀도를 높인다.

## 연결
- 구체화 대상 URS: URS-005(책임감), URS-003(함께 운동)
