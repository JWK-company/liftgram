## 1. 가입 · 자격 · 앱 허용 여부

### 결론
**개인·개인사업자·법인 모두 가입 가능하고, 모바일 앱은 공식적으로 허용되는 활동 매체다.** 단 가입은 2단계(계정 생성 → "최종승인")이며, 최종승인 전에는 정산도 API도 불가하다.

### 근거
- 운영정책 1.1 가입심사: *"쿠팡 회원이라면 누구나 쿠팡 파트너스 서비스 신청이 가능합니다. **단, 미성년자나 비영리법인, 면세사업자의 경우 신청이 거절될 수 있습니다.**"* / *"서비스 신청 이후 회사가 사전에 안내한 일정 금액의 수익 발생 시 해당 회원의 채널을 검토하여 최종승인 여부를 결정합니다."*
- 이용약관 4.1.2 나: 승낙 거절 사유 — 쿠팡 회원이 아닌 경우, 쿠팡 임직원·직계가족, 과거 자격상실 이력, 이용제한 중 재가입, 중복가입, 타인 명의 등.

## 2. 수수료율 · 캡 · 쿠키(전환 인정 기간)

### 결론
**요율은 카테고리·상품별로 쿠팡이 정하되 별도 지정이 없으면 결제금액의 3%.** 스포츠/레저·헬스용품에 대한 **별도 우대 요율은 공식 문서에 존재하지 않으며 사실상 3%로 봐야 한다.** 캡은 **월 3,000만원**(초과분 미지급). 전환 인정은 **클릭(접속) 시각 기준 24시간 라스트클릭**.

### 근거
- 운영정책 5. 수익배분 (현행 2026.6.8판): *"구매상품의 수익 지급율 및 지급제외상품 — **수익 지급율은 카테고리 및 상품별 특성을 고려하여 회사가 정한 바에 따름. 단, 별도로 정하지 않은 경우에는 결제금액의 3%로 함.**"* / *"기프트카드 및 유상으로 제공되는 제품 보증서비스, 기타 회사가 사전 공지한 상품/카테고리의 경우, 수익금 지급 대상에서 제외됨."* / *"수익 지급율 및 지급제외 상품은 회사의 사정에 따라 변경될 수 있음."*
  - (참고) 직전 버전 운영정책에는 *"**모든 카테고리 결제금액의 3.00%** (단, 기프트카드는 제외)"*로 더 명시적이었다.
- 이용 가이드 PDF(2025) STEP 5: 실적 제외 = 기프트카드, 유상 제품보증서비스(쿠팡안심케어·AppleCare+). **다른 요율 적용 상품 = 삼성·애플 스마트폰 1%**(액세서리는 기존과 동일).
- 캡: FAQ「수익 & 정산 > 수익금 한도가 정해져 있나요?」 *"파트너스 회원의 월 수익금 최대 한도는 **월 3천만원**입니다. (2024.8.1부터 적용)"* / 가입화면 문구 `REGISTRATION_COMMISSION_CAP_TIPS`: *"…최대 금액은 1개월 기준 3,000만원이며, **이를 초과하여 발생한 수입은 이월되거나 지급되지 않습니다.**"*
- 쿠키: FAQ「어떤 경우에 수익이 발생하나요?」 *"회원님께서 생성하신 링크/배너 등으로 고객이 접속하여, **접속 시간 기준 24시간 내에 발생한 모든 구매액**에 대해 수익이 집계됩니다."*
- 라스트클릭: 가이드 PDF STEP 5 *"여러 사람의 링크로 클릭이 발생한 경우, **고객이 가장 마지막으로 접속한 경로가 회원님의 링크일 때** 실적 집계가 가능해요."*
- 정산: 최소 지급 1만원 / 실적월 익월 25일 확정 → **익익월 15일 지급** / 미달 시 최대 6개월 이월 (운영정책 5, 이용약관 4.1.3다).

### 확신도
- 3% 기본요율 · 24시간 쿠키 · 월 3천만원 캡 · 정산 스케줄: **확실**
- "스포츠/레저 5~10%" 류의 블로그 주장: **근거 없음(오정보)**. 카테고리별 우대 요율표는 공식 문서상 **확인 불가**(쿠팡이 "회사가 정한 바"로만 규정하고 표를 공개하지 않음).

### Liftgram 함의
리스트랩·스트랩(1~3만원대) 기준 건당 수수료는 **300~900원 수준**. 벨트(5~15만원)도 1,500~4,500원. **객단가가 낮아 순수 수수료만으로는 수익 모델이 성립하기 어렵다** — 볼륨 아니면 "부가 기능" 포지션으로 봐야 한다.

## 3. 오픈 API — 존재 여부 · 승인 조건 · URL 파라미터 대안

### 결론
**존재한다. 단 "최종승인"(누적 판매금액 15만원) 이후에만 키가 발급된다.** 상품검색 API는 별도 승인 절차 없이 최종승인만 통과하면 무료로 쓸 수 있고, **분당 50회** 제한이다. 최종승인 전 사용은 **메일 심사라는 예외 경로**가 공식적으로 열려 있다. **API 없이 검색 URL에 추적 파라미터만 수동으로 붙이는 방식은 공식 지원 방식이 아니며, 약관상 "광고 링크 조작" 금지에 걸릴 위험이 있다.**

### 근거
- **최종승인 요건**: FAQ「최종승인 > 최종 승인은 어떻게 받나요?」 *"**누적 판매 금액이 15만원 이상 발생하면** 채널 검토 후 최종 승인이 이루어집니다."* / 「최종 승인이 뭔가요?」 *"최종 승인이 완료되어야 쿠팡 파트너스 회원으로써 **파트너스 API 기능이용 및 첫 정산**이 가능합니다."*
- **예외 경로 (중요)**: OPEN_API_GUIDE 원문 *"파트너스 API는 최종 승인 된 쿠팡 파트너스 회원에게만 제공됩니다. **최종 승인 전 파트너스 API 사용이 필요하신 경우 `AF ID와 함께 광고를 게재할 영역과 광고형태`에 대해 파트너스 메일로 보내주시면 검토 후 안내 드리겠습니다.**"*
- **비용**: OPEN_API_FAQ *"아니요, 무료입니다. 파트너스 API는 최종 승인된 파트너스 회원이라면 누구나 사용 할 수 있습니다."*
- **엔드포인트** (base `https://api-gateway.coupang.com/v2/providers/affiliate_open_api/apis/openapi/v1/`):
  | 메서드 | 경로 | 비고 |
  |---|---|---|
  | POST | `/deeplink` | 쿠팡 URL → 파트너스 단축 URL. **최대 20개 배치**, `subId` 지원 |
  | GET | `/products/search` | 키워드 검색. **1분당 최대 50회**, `limit` 최대 10개, `subId`·`imageSize`·`srpLinkOnly` 지원 |
  | GET | `/products/bestcategories/{categoryId}` | 카테고리 베스트, limit 최대 100 |
  | GET | `/products/goldbox`, `/products/coupangPL`, `/products/coupangPL/{brandId}` | |
  | GET | `/products/reco` | ADID 기반 개인화 추천 (v2 `products/reco`도 존재) |
  | GET | `/reports/clicks|orders|cancels|commission` | 실적 리포트, 매일 15:00 갱신 |
- **인증**: HMAC-SHA256 CEA 서명 (`Authorization: CEA algorithm=HmacSHA256, access-key=…, signed-date=…, signature=…`), 서명 메시지 = `datetime + METHOD + path + query`. 키는 **재발급/삭제 불가**(OPEN_API_FAQ).
- **레이트리밋 위반 시**: OPEN_API_FAQ *"API 호출 횟수 제한에 도달한 경우 **API 사용이 24시간 동안 제한**됩니다."* (구판 PDF 기준: 1회 위반 24시간 차단, 3회 위반 시 모든 파트너스 기능 차단)
- **딥링크가 지원하는 URL 형식** (DeepLinkBody 정의 원문): 쿠팡 홈 / 제품 상세 `/vp/products/{id}` / **검색 결과 `https://www.coupang.com/np/search?component=&q=toys&channel=user`** / 기획전 / 골드박스 / 로켓와우 / 로켓배송 / 로켓직구 / 여행상품. → **검색 URL 딥링크화는 공식 지원된다.**
- **수동 파라미터 부착에 대한 반대 근거**: 운영정책 4.1 1) 기술적 금지 행위 — *"**자사가 전송하는 광고의 링크, 형태, 갱신주기 및 이에 포함된 정보에 대한 별도의 조작 행위**"* 금지. 공식 링크 생성 경로는 (a) 웹 링크 생성기/간편링크, (b) Deeplink API 두 가지뿐이며 파라미터 규격은 공개되지 않았다.

### 확신도
- API 존재·최종승인 전제·무료·엔드포인트·레이트리밋: **확실**
- 최종승인 전 메일 예외 심사의 실제 승인 기준(트래픽·매출 요건): **확인 불가** (쿠팡이 기준을 공개하지 않음, "검토 후 안내")
- 수동 `lptag` 파라미터 부착 허용 여부: **불확실 → 사실상 불허로 취급 권장** (공식 문서에 허용 근거 없음 + 링크 조작 금지 조항 존재)

### Liftgram 함의 (매우 중요한 3가지 제약)
1. **닭-달걀 문제**: API 키를 받으려면 먼저 누적 15만원 매출이 필요한데, 우리 앱의 태그 기능 자체가 API를 필요로 한다. → 초기에는 **웹 링크 생성기로 수동 생성한 딥링크를 큐레이션 DB에 넣는 방식**으로 시작하고, 15만원 돌파 후 API로 전환하는 2단계 설계가 필수. (또는 메일 예외 심사 선요청)
2. **API 데이터 저장 금지 조항**: 오픈 API 약관 제5조 3항 — *"이용자는 회사가 제공하는 서비스 이용과 관련하여 API 서비스를 통해 제공된 데이터에만 접속할 수 있으며, **API 서비스를 통해 제공된 해당 데이터를 복제, 저장 또는 전송할 수 없습니다.**"* → **상품명/가격/이미지를 우리 DB에 캐싱하는 설계는 약관 위반 소지.** 분당 50회 제한 하에서 캐싱 없이 서비스하려면 아키텍처가 크게 제약된다. (실무상 널리 캐싱되고 있으나, 조항은 존재한다 — 리스크로 명시할 것)
3. **인증키 1개 제한**: 오픈 API 약관 제4조 1항 — *"이용자는 1개의 인증키를 받아 사용할 수 있으며, 어떠한 경우에도 2개 이상의 인증 키를 가지거나 발급받으려고 해서는 안됩니다."* 제2항 — *"자신의 인증키를 **제3자에게 제공, 공개, 공유, 대여 또는 양도할 수 없습니다.**"* → 사용자별 키 발급 구조는 불가. **앱 사업자 단일 계정 모델만 가능.**

## 5. 수수료 재분배 (사용자에게 현금/포인트/크레딧 지급) — 최우선 확인 항목

### 결론
### **금지된다. 명시적 조항이 두 곳에 존재한다. 이 계획은 "우리가 받은 수수료를 사용자에게 나눠주는" 형태로 설계하면 안 된다.**

### 근거 (원문 인용)

**(1) 이용약관 제7조(회원의 의무) 제2항 — 금지행위 목록**
> **"방문자에게 금품을 제공하는 등 기타의 방법으로 쿠팡보다 더 낮은 가격으로 상품을 구매할 수 있도록 유인하는 행위."**

→ 캐시백·포인트·크레딧은 실질적으로 "쿠팡보다 낮은 실구매가"를 만드는 금품 제공이다. **캐시백 금지 조항이 바로 이것이다.**

**(2) 운영정책 2.2 미디어 등록제한 — 등록 자체가 거절/취소되는 사유**
> **"광고 시청, 클릭 등의 콘텐츠 이용에 대해 보상을 제공하는 경우"**
> **"방문자에게 금품을 제공하는 등 기타의 방법으로 쿠팡보다 더 낮은 가격으로 상품을 구매할 수 있도록 유인하는 행위"**
> **"비정상적으로 트래픽을 유도하는 경우"**

→ 리워드형(광고 보면 포인트) 매체는 **최종승인 이전 단계인 "미디어 등록"에서부터 거절**된다.

**(3) 이용약관 제7조 — 계정/권한 재판매 금지**
> "계정을 매매, 양도, 명의변경, 재 판매하거나 질권의 목적으로 사용하는 행위"
> **"서비스 이용 권한을 타인에게 공유, 제공, 양도, 중개, 재 판매하는 행위."**

**(4) 운영정책 4.1 5) 광고미디어 권한의 재판매 행위**
> "회원은 당사로부터 부여 받은 광고미디어로서의 권한을 **회사의 서면 동의 없이** 타인 또는 타 매체에 공유, 제공, 양도, 중개, 재 판매 등을 할 수 없습니다."

→ "사용자에게 파트너스 수익을 쉐어해 줄 테니 링크를 걸어라"는 구조는 **사실상 우리 파트너스 권한을 제3자(사용자)에게 재판매/중개하는 것**으로 해석될 수 있다.

**(5) 참고 — 쿠팡이 인정하는 유일한 "일반인 리워드" 경로**
이용약관 제2조·제4.2조에 **"리워드 프로그램"**이 별도로 존재한다: *"쿠팡 모바일 앱 내 공유하기 기능을 통해 … 회사가 얻은 수익을 회사와 회원이 일정한 비율로 분배하는 광고서비스"*, 정산은 **쿠팡캐시**로, **결제금액 기준 월 최대 50,000원**. 즉 개인 사용자에게 보상을 주는 구조는 **쿠팡이 자기 앱 안에서 직접 운영**하며, 제3자 앱이 이를 대행하는 경로는 없다.

### 확신도
**확실** — 명시적 조항 2건(약관 제7조 + 운영정책 2.2) 직접 확인.

### Liftgram 함의 (설계 분기)
- ❌ **불가**: "네 사진에 태그한 리스트랩이 팔리면 너에게 포인트 준다" (= 방문자/사용자에게 금품 제공 + 클릭 보상 + 권한 재판매)
- ⚠️ **회색**: 구매와 무관한 순수 활동 리워드(운동 기록 스트릭 포인트 등)와 파트너스를 **완전히 분리**해 운영. 단 "쿠팡 링크 클릭/구매"와의 인과가 조금이라도 연결되면 위반.

- ✅ **가능**: 수수료를 사용자 개인이 아니라 **서비스 운영비/무료 기능 유지**에 쓴다고 고지 (개별 유인 아님).

### D. 권장 로드맵

```
Phase 0-a (지금)        : A1~A9 구현 + 카탈로그 20종 큐레이션 (딥링크 컬럼 비움)
Phase 0-b (앱 출시 직후) : B1~B6 처리 → A3에 수동 딥링크 주입 → 기능 ON
Phase 0-c (병행)        : B9 사전 협의 메일 발송 (UI 목업 첨부), B8' API 예외 심사 요청
Phase 1  (15만원 달성 후): B7·B8 → Deeplink API로 링크 생성 자동화
                          (products/search는 분당 50회 + 저장 금지 제약 하에서만 제한적 사용)
Phase 2  (확장)         : 링크프라이스 등 병행 네트워크로 브랜드 직판 커버
```

### E. 최대 리스크 3가지

1. **수익성**: 기본 3% × 리스트랩 2만원 = **건당 600원**. 월 100만원 수익을 내려면 월 3,300만원 거래액이 필요하다. Phase 0을 "수익 모델"이 아니라 **"장비 정보 UX"**로 포지셔닝하고, 수수료는 부수입으로 잡는 게 현실적이다.
2. **UI 정책 리스크**: 사진 위 태그가 2026.6.8 신설된 "커버형/가림 광고" 해석에 걸리면 **A등급 1회 = 최근 14일 수익금 몰수, 2회 = 계정 해지**. 게다가 **연관 계정 전체로 확산**되고 **경쟁자 제보 포상금 5만원 제도**가 가동 중이다. → B9 사전 협의를 Phase 0-c가 아니라 **Phase 0-a와 병행**으로 앞당길 것을 권장.
3. **닭-달걀**: API 키 = 최종승인 = 누적 15만원 매출. 유기적 매출 없이는 자동화 불가. → **수동 큐레이션 카탈로그로 시작하는 설계가 선택이 아니라 필수.**
