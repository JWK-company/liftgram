## 문서별 핵심 요약

### 1. health-app.md — 헬스앱 프로젝트 정의
- 단계: **discovery**. 정의/타겟/핵심가치 모두 빈칸(미확정).
- 백그라운드 체크 종합: **Pivot (컨셉 강제 협소화), Go 불가.**
- 경쟁: Hevy·Fitbod·Strava·플랜핏·콰트·**삼성헬스(OS 번들 무료 = 위협)**.
- 틈새: 시니어 낙상예방·재활(의료법벽), 여성건강, **"기록 × AI 자세분석 통합"**(통합 승자 부재).
- 법: 디지털의료제품법/의료기기법 — 진단·치료·예방 표방 시 식약처 허가. **"웰니스" 포지션 유지 필수.**
- 수익화: 프리미엄 전환율 ~2% vs 하드페이월 12%. **D30 리텐션 3~10%**(구조적 이탈).
- 기술: 순수 기록=차별화 0. AI 자세분석=중상 난이도 + 부상 책임. **Google Fit API 종료→Health Connect**, 초기 **iOS HealthKit 집중** 권장.

### 2. hevy-success-analysis.md — 벤치마크 + 차별화 (가장 중요)
- Hevy: 부트스트랩으로 **1,400만 유저**. 성공요인 6 (소셜 빈칸, 스코프 규율 3기둥, 무료·무광고, SEO 엔진, 커뮤니티 주도, **"피트니스" 아닌 "웨이트/근력" 니치**).
- **차별화 5 wedge**: A 지능형 프로그래밍/주기화(1순위) · B 한국 로컬라이즈 · C 좁은 수직 1개 · D 엑셀 자산 마이그레이션 다리 · E 카운팅/자동입력.
- 3앱(Hevy/Fitbod/Planfit) 포지셔닝 비교 + **강점 합성 레시피**(진짜 빈칸 = 능동적 책임감 / 플라이휠).

### 3. 20260331_workout_excel_builder.md — 솔로 검증 원본
- "운동인 커스텀 엑셀 빌더" 아이디어 분석 **HOLD 4.2/10** (수익화 3점이 발목).
- 기능 아이디어와 차별화 wedge(엑셀 자산 마이그레이션)의 출처.
- ⚠️ 팀의 "헬스앱"이 이 엑셀 빌더와 다른 컨셉이면 별도 검증 필요.

### 4. risks.md — 착수 전 IP·법 게이트
- **최우선: 본업 회사 IP 귀속.** 경업(non-compete) OK여도 IP 귀속은 별개 리스크.
  - 근로계약 발명양도·직무발명 조항 확인 / 회사 자산·시간 미사용 / 근무시간 외 증빙.
- 개인정보(건강=민감정보) 처리, 미등록 수익 세무.
- ※ 동업 거버넌스 성격이 일부 섞여 있음 — 순수 개발 문서만 원하면 제외 가능.
