# Hevy 성공 분석 + 차별화 전략 — 발표용

> 직접 경쟁자이자 벤치마크. 팀 미팅 발표 자료.
> 실행 트래커는 [health-app.md](../projects/health-app.md), 비교는 [project-scan.md](project-scan.md), 솔로 검증은 [엑셀 빌더 idea](../../04-side-projects/idea/20260331_workout_excel_builder.md).

## 1. 한눈에 보는 숫자

| 지표 | 값 |
|------|----|
| 출시 | 2019.11 (Guillem Ros + Desmond McNamee, 바르셀로나) |
| 유저 | **1,400만+** (별점 4.9, 리뷰 50만+) |
| 최근 월 지표 | 다운로드 ~40만 / 매출 ~$60만 (2022년엔 월 $2만 → 30배) |
| 자금 | **부트스트랩** (VC 없이 소수 팀이 키움) |
| 가격 | 무료 + 광고 없음 / Pro 월 ~$8.99·연 ~$23.99 |
| 웹 트래픽 | 월 ~72.5만 방문, **77%가 SEO(검색) 유입** |

- **핵심 한 줄**: "세상에서 가장 단순하고 직관적인 운동 기록 앱" + **소셜**.
- Bible Chat이 *VC로 escape velocity*였다면, Hevy는 **부트스트랩 + SEO + 소셜 네트워크 효과**의 정반대 모범.

## 2. 핵심 기능 / 특장점 (= 엑셀 빌더로 하려던 것 거의 전부)

| 기능 | 설명 |
|------|------|
| **루틴 빌더** | 커스텀 운동 루틴 구성·저장·반복 (네가 만들려던 빌더의 앱 버전) |
| **세트 로깅** | 무게·횟수·RPE 빠른 기록, 직전 기록 자동 표시 |
| **대체 운동** | 기구 상황에 맞춰 즉석 교체 (네 "대체 운동 토글"과 동일) |
| **분석/대시보드** | 볼륨·1RM 추정·진척 그래프·PR 추적 |
| **소셜** | 친구 팔로우·운동 피드·리더보드·좋아요 (차별화의 핵심) |
| **무광고·관대한 무료티어** | 진입장벽 0 → 네트워크 효과 극대화 |
| **Hevy Coach** | 트레이너용 B2B 확장 (PT가 고객 프로그램 관리) |

> ⚠️ **그래서 "엑셀 빌더" 정면 컨셉은 죽음.** 네 솔로 검증이 차별화 3/10·수익화 3/10이었던 이유가 이것 = Hevy가 이미 다 함 + 무료. **"엑셀 파일이 나온다"는 포맷 차별화이지 가치 차별화가 아님.**

## 3. 왜 성공했나 — 6가지

### ① 커머디티(기록) 위에 진짜 빈칸(소셜)을 얹었다
- 당시 기록 앱은 많았지만 **친구와 비교·격려하는 소셜이 없었다** (사이클 앱엔 있는데 헬스엔 없던 갭).
- 창업자 인사이트: "친구 운동을 보고 같이 동기부여 받고 싶다."
- 교훈: **기록은 이미 상품화됨 → 차별점은 그 위의 레이어(소셜/책임감)에서.**

### ② 스코프 규율 — 3개 기둥만
- 초기에 과설계로 "다 되지만 다 어설픈" 앱을 만들었다가, **로깅·분석·소셜 3개로 잘라냄.**
- 교훈: 기능 부풀리기 = 죽음. **핵심 3개에 집중해야 단순함이 무기가 됨.**

### ③ 무료 + 무광고 → 네트워크 효과 연료
- 소셜은 **규모가 있어야 작동** → 과금/광고로 진입을 막지 않음.
- React Native 크로스플랫폼으로 iOS/Android 동시 = 소셜 도달 최대화.
- 교훈: 네트워크형 제품은 **초반엔 수익보다 도달**.

### ④ SEO 콘텐츠 엔진 = 저비용 유저 획득
- 웹 트래픽 **77%가 검색** ("best workout tracker", 운동별 가이드 등).
- 페이드 마케팅 거의 없이 **CAC를 구조적으로 낮춤** → 부트스트랩으로 생존 가능.
- 교훈: 신앙앱(Bible Chat)=숏폼 바이럴, 헬스(Hevy)=**SEO**. 도메인마다 분배 채널이 다름.

### ⑤ 부트스트랩 + 커뮤니티 주도 개선
- 소수 팀이 유저 피드백으로 빠르게 반복 → **"좋은 제품이 최고의 마케팅".**
- 교훈: 우리 3인 팀 구조와 가장 닮은 모델. VC 없이도 가능함을 증명.

### ⑥ 니치 집중 — "피트니스"가 아니라 "웨이트/근력"
- 칼로리·홈트·러닝 다 하는 만능앱이 아니라 **근력운동 기록에 특화.**
- 교훈: **좁혀서 그 안에서 1등.** (= 우리 백그라운드 체크의 "수직 1개 협소화" 권고와 동일.)

## 4. 카피하되 우리만의 차별점 두는 법 (핵심)

**원칙**: Hevy의 *기록·분석·소셜*은 커머디티 → **그대로 베끼면 0승.** 차별화는 **Hevy가 일부러 안 하는 곳**에서 나온다.

> Hevy의 공식 철학: *"우리는 어떻게 운동할지 알려주지 않는다 — 도구만 준다(hands-off)."*
> ⇒ **바로 그 빈칸이 우리 자리.** 너의 "엑셀 빌더"의 본질(프로그램을 *설계*해주는 것)이 여기에 정확히 들어맞는다.

### Wedge A — 지능형 프로그래밍/주기화 (1순위 추천)
- Hevy = 로거(기록만). **프로그램을 짜주진 않음.**
- 우리 = **AI가 목표·경력·장비·일정 입력 → 점진적 과부하(progressive overload) 프로그램 자동 생성 + 주차별 자동 진행/디로드.**
- 네 엑셀 빌더 아이디어를 **"수동 빌더"가 아니라 "코치형 자동 빌더"로 재정의** → 포맷이 아닌 **가치 차별화**. Fitbod가 근접하나 기록(Hevy)과 분리됨 → **기록+코칭 통합 승자 부재**(우리 빈칸).

### Wedge B — 한국 로컬라이즈
- 한국 **헬스장 기구 기반** 루틴(플랜핏 강점) + 한국어 운동 DB + 한국 헬스 커뮤니티 문화(오운완 인증 등).
- 글로벌 Hevy가 약한 지점. 단 국내도 경쟁(플랜핏 400만·콰트) → **로컬만으론 부족, A/C와 결합 필수.**

### Wedge C — 좁은 수직 1개
- 예: **파워리프팅/스트렝스 주기화 전문**, 또는 **시니어 근감소증 예방**, 또는 **단일 종목 폼교정.**
- "모두를 위한 헬스앱"은 Hevy에 짐. **한 부족에게 압도적인 앱**이 되는 게 승산.

### Wedge D — 마이그레이션 다리 (네 엑셀 자산 활용)
- 이미 **엑셀/구글시트로 운동 기록하는 사람**을 타깃: "네 시트 import → 앱으로 자동 전환."
- 엑셀 빌더 아이디어의 유일한 진짜 진입점(TAM은 좁지만 명확). 단독 제품 아닌 **온보딩 기능**으로.

### Wedge E — 카운팅/자동 입력 (사용자 메모 반영)
- 센서/카메라 기반 **자동 횟수 카운팅** → 로깅 마찰 제거(Hevy도 수동 탭).
- 단 정확도·종목 커버·부상 책임(의료 리스크) → 좁게. **웰니스 포지션 유지**(진단·치료 표방 금지, 식약처벽 회피).

### 추천 조합
> **A(지능형 프로그래밍) × C(좁은 수직 1개) × B(한국)** 를 묶고, D를 온보딩, E를 후속 기능으로.
> 즉 **"○○(좁은 대상)를 위한, 프로그램을 짜주는 한국형 헬스 코치"** = Hevy(로거)와 정면충돌 안 함.

## 5. 3개 앱 포지셔닝 비교 (Hevy vs Fitbod vs Planfit)

> **결론 먼저: 셋은 같은 카테고리지만 포지션이 확연히 다르다.**
> 축은 **로거(기록만) ↔ 플래너/코치(뭘 할지 정해줌)**.

| | **Hevy** | **Fitbod** | **Planfit (플랜핏)** |
|---|---|---|---|
| 한 줄 | 기록 + **소셜** 로거 | **AI 적응형 추천 엔진** | **AI 플래너 + 한국 로컬** |
| 철학 | "어떻게 운동할지 안 알려줌, 도구만" | "오늘 뭘 할지 정해줌"(회복·과부하 기반) | "초보도 PT처럼"(기구·자세까지) |
| 핵심 강점 | 단순 UX·**소셜/피드/리더보드**·무료 | 회복 인지 추천·1,600+ 종목·과학 알고리즘 | **헬스장 기구 기반 추천·3D 자세 가이드·한국어** |
| 프로그램 생성 | ✕ (네가 짜야 함) | ◎ 자동·매 세션 적응 | ◎ 자동 + 기구/초보 맞춤 |
| 자세/폼 가이드 | △ | △ | ◎ **3D 애니메이션** |
| 소셜 | ◎ | ✕ | △ |
| 타깃 | 중급+ 웨이트, 글로벌 | 홈/짐 리프터, 글로벌 | **초보·한국 헬스장 이용자** |
| 가격 | 무료 + Pro 월 ~$8.99 | 월 $15.99 / 연 $95.99 (무료X) | **무료 티어 강함** + ChatGPT AI코치 프리미엄(≈월 2만원 PT 대체) |
| 규모/배경 | 1,400만, 부트스트랩, SEO | 4.81★ 27만 리뷰, 글로벌 강자 | **400만, 한국**, 운동데이터 8천만+ 학습 |

**핵심 차이 3줄**
1. **Hevy = "기록·소셜"** — *너가* 프로그램을 짜고, 앱은 로깅·분석·자랑을 도움. 프로그램 생성은 안 함.
2. **Fitbod = "추천 엔진"** — 회복 상태·과부하를 보고 *오늘의 세션*을 자동 구성. 소셜 없음, 비쌈, 글로벌.
3. **Planfit = "플래너 + 한국 로컬"** — Fitbod의 추천 + **헬스장 기구 기반 + 3D 자세 + 초보 친화 + 한국어**. 무료 진입.

> ⚠️ **차별화 전략 재조정(중요):**
> 4절의 **Wedge A(지능형 프로그래밍)·B(한국 로컬)는 이미 Planfit이 상당 부분 점유** 중이다(=백그라운드 체크의 Pivot 경고가 맞았음).
> 즉 "AI가 한국 헬스장 루틴 짜줌"만으론 **Planfit 정면충돌**. 진짜 빈칸은 더 좁혀야 함:
> - **셋의 교집합 빈칸**: *Fitbod의 적응형 코칭* + *Hevy의 소셜/책임감* 을 **한 앱에서** (Fitbod엔 소셜이 없고, Hevy엔 코칭이 없음) → **"같이 하는 AI 코치"**.
> - 또는 **Planfit이 약한 수직**(Wedge C): 시니어·재활·여성·파워리프팅 주기화 등 좁은 부족.
> - **결론: A·B 단독은 죽음 → C(좁은 수직) × "소셜+코칭 융합" 으로 다시 좁혀야 함.**

## 6. 우리 팀 시사점

1. **Hevy가 다 한다 = 수요 검증됨(좋은 신호)**, 그러나 **정면 카피는 차별화 0 = 죽음.**
2. **부트스트랩 + SEO + 소셜**은 우리 3인 구조에 가장 현실적인 모델 (VC 불필요).
3. **분배 채널은 SEO** (Bible Chat의 숏폼과 다름) → 콘텐츠/검색 전략을 빌드 초기부터.
4. **차별화는 "Hevy가 안 하는 것"** = 프로그램을 *짜주는* 코치 레이어 + 좁은 수직 + 한국.
5. ⚠️ **의료기기벽**: AI 코칭/카운팅은 "웰니스"로 포지션, 진단·치료·"질병 개선" 표현 금지(식약처).

## 7. 추천 조합 레시피 — 3앱 강점 합성

> **원칙: 기능을 더하지 마라. 더하면 = Planfit 재구현 = 3인 팀이 정면으로 짐.**
> **합성할 것은 "기능"이 아니라 셋 중 누구도 못 만든 "플라이휠"이다.**

### 진짜 빈칸 = 능동적 책임감(active accountability)
- 헬스앱 공통 약점: **D30 리텐션 3~10%** (사람들이 그냥 그만둠).
- Fitbod·Planfit의 코칭은 "뭘 할지"는 풀지만 **"왜 계속 하는지"는 못 품**.
- Hevy 소셜은 피드·리더보드 = **수동적 "자랑"**이지, 빠지면 끌어주는 **능동적 책임감이 아님.**
- ⇒ **누구도 안 가진 빈칸 = "같은 목표의 사람과 함께, 빠지면 챙겨주는 구조".** 이게 리텐션 엔진이자 성장 엔진(추천·그룹 바이럴).

### 각 앱에서 가져올 것 / 버릴 것

| 앱 | 가져온다 ✅ | 버린다 ✂️ |
|----|-----------|----------|
| **Fitbod** | 회복 인지 **적응형 코칭**(오늘 뭐 할지·과부하 자동) | 고가 페이월·소셜 부재·범용 글로벌 |
| **Hevy** | **단순 로깅 UX** + **무료+SEO 성장** + 소셜의 뼈대 | 수동적 피드(자랑용)·중급자 전제 |
| **Planfit** | **3D 자세 가이드** + 초보 친화 + 헬스장 기구 매핑 | "한국 범용 코치" 포지션(=Planfit과 정면충돌) |
| **(신규)** | **능동적 책임감 루프**(운동 메이트·그룹 챌린지·결석 케어) | — 셋 다 없음 = 우리 해자 |

### 플라이휠 (이게 경쟁력)

```
좁은 부족(tribe) 선정
  → AI 코칭이 "오늘 뭘 할지" 마찰 제거 (Fitbod+Planfit)
  → 같은 부족 메이트와 함께/챌린지 = 빠지면 끌어줌 (신규 책임감)
  → 리텐션↑ → 부족 안에서 소셜 밀도↑ → 추천으로 새 멤버
  → SEO 콘텐츠로 부족 유입 (Hevy식 저비용 획득)
  → 다시 처음으로
```
- **핵심: 소셜은 "넓게"면 죽는다(텅 빈 피드). "좁은 부족"이라야 작은 유저로도 꽉 찬 느낌** → Hevy도 초기엔 작은 운동 커뮤니티들로 시작.

### 부족(vertical) 후보 — 소셜이 자연 발생하는 곳으로

| 후보 | 왜 | 경쟁 노출 |
|------|----|----------|

### 한 줄 포지셔닝 (예시)
> **"○○(부족)를 위한, AI가 짜주고 동료와 함께 지키는 운동 앱"**
> = Fitbod(코칭) + Hevy(소셜·무료) + Planfit(3D·초보) 의 강점을 **한 부족 안에서** 합성. 범용 3앱과 정면충돌 안 함.

### 수익화 (3앱 교훈 합성)
- **무료 진입**(Hevy·Planfit) — 소셜·SEO 연료 위해 진입장벽 0.
- **프리미엄 = AI 코치 심화**(Planfit·Fitbod) — 개인화 코칭·고급 분석.
- **그룹/챌린지·PT 마켓**(신규) — 책임감 루프를 유료 그룹·트레이너 연결로 수익화(Hevy Coach 변형).
- ⚠️ 헬스앱 무료→유료 전환율 중앙값 2.18% → **하드페이월보다 "그룹 가치"로 묶는 리텐션 우선.**
