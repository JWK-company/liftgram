# 설문(survey) 보고서 가이드 — 의사결정 수집용

> 용도: 사용자에게 **여러 결정을 한 번에 받아야 할 때**(요구 접수 후 설계 갈림길, 우선순위 확정, 회고 설문). `_COMMON.md` 선행 숙지 — 특히 §3(sandbox 제약)이 이 타입의 핵심 제약.

## 구조 (위→아래)
1. **헤더**: 제목 · 문항 수 · "상세 배경은 첨부 전체 보고서 참조" 링크 문구.
2. **작성 방법 박스**: ①선택→의견 ②[응답 모아 복사] 버튼 ③메신저에 붙여넣어 전송 ④입력값은 페이지 닫으면 소실(한 번에 작성) ⑤빈 문항은 "미응답" 허용.
3. **진행 카운터**(sticky): `n / 총문항` + 진행 바. change/input 이벤트로 갱신.
4. **문항 카드** (섹션별 그룹핑 A/B/C…):
   - 문항 ID(Q1, Q3-부속① 등 — **응답 텍스트의 키가 되므로 안정적으로**) + 질문 한 줄.
   - 배경 1~2줄(길면 `<details>` 접이식으로).
   - **예시 답변을 입력칸 위에**(보라 박스 — 사용자가 어떤 형식·수준으로 답하면 되는지 즉시 알게).
   - 입력: 선택지는 radio(라벨에 1줄 장단점, 추천엔 "— 추천" 표기), 자유 의견은 textarea. 정보 요청 문항(로그 붙여넣기 등)은 textarea만.
5. **자유 의견 문항**을 마지막에 반드시 1개.
6. **수집기(fixed 하단 바)**: [📋 응답 모아 복사] 버튼.

## 수집기 JS 계약 (sandbox 안전 패턴 — 이대로 구현)
- 각 문항 컨테이너에 `data-item="<ID>"`, 의견 textarea에 `data-note="<ID>"`.
- 버튼 클릭 → `"<ID>: <선택값|(의견만)|미응답> — <의견>"` 형식 줄들을 헤더 라인과 함께 조립 → readonly textarea에 넣고 `select()`.
- 복사: `navigator.clipboard.writeText` 시도 → 실패 시 `document.execCommand("copy")` → 둘 다 안 되면 "전체 선택됨 — Ctrl+C" 상태 표시. **try/catch 필수**.
- `localStorage`/`document.cookie` 절대 사용 금지(opaque origin에서 예외).
- 폼 `submit` 사용 금지 — 반환 채널은 항상 "복사→메신저 붙여넣기".

## 응답 처리 (세션 측)
- 사용자가 붙여넣은 응답은 `<ID>: <값>` 줄 단위라 기계 파싱 가능 — 후속 작업(기획 개정·발급)에 그대로 매핑.
- "미응답" 문항은 후속 질문으로 재수집(전체 재설문 금지).

## 검증 추가 항목 (C3에서)
- **실제 브라우저로**: 문항 수 렌더 확인 → radio 1개 선택+textarea 입력 → 카운터 증가 → 수집 버튼 → 출력 텍스트에 선택·의견 포함 확인.
- 예시 답변이 모든 문항에 있는지, 문항 ID가 응답 시트/후속 파서와 일치하는지.
