# 쇼케이스(showcase) 보고서 가이드 — 기능 소개·데모

> 용도: 완성된 기능을 **보여주는** 보고서(신기능 공개, 마일스톤 데모, 사용 가이드). 청중은 기술 배경이 없을 수 있음. `_COMMON.md` 선행.

## 핵심 원칙
- **화면이 주인공**: 스크린샷·GIF가 서사를 끌고, 텍스트는 캡션·맥락만. 각 이미지에 "여기서 무엇을 보라"는 캡션 필수.
- **사용자 여정 순서**: 기능 나열이 아니라 실제 사용 흐름(로그인→작업→결과) 순.
- **전문용어 최소화**: 내부 코드명 대신 화면에 보이는 이름으로.

## 섹션 구성
1. **헤더 + 한 줄 소개** — 이 기능이 사용자에게 뭘 해주는가.
2. **하이라이트 3~5개** — 카드: 이미지 + 2줄 설명.
3. **사용 흐름** — 단계별(①②③…) 화면과 조작 설명.
4. **세부 기능 표** — 짧게(이름·설명·위치).
5. **알려진 제약·다음 계획** — 솔직하게 1절.
6. **부록** — 버전·환경·문의 경로.

## 미디어 규약
- 이미지·GIF는 `plm_upload.py`로 올려 `/api/files/<key>` 참조(첨부 열람과 같은 게이트) 또는 data URI 인라인(200KB 이하 권장).
- GIF는 5초 내외·핵심 조작만. 떨림·과한 줌 금지(실조작 녹화 권장 — 실측 교훈).
- 다크 화면 스크린샷은 라이트 인쇄에서 뭉개지지 않게 테두리 처리.
